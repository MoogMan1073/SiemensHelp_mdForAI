### 24 V voltage supply

Connect a direct voltage of 24 V to the L+ and M terminals for the FM 350-2 voltage supply.

### 8.2 V DC encoder power supply

The module generates a voltage of 8.2 V (max. 200 mA) from the 24 V supply voltage. This voltage is available at the terminals P8V2 (pins 19, 20, 39, and 40) for the voltage supply to the NAMUR encoders and is resistant to short circuits.

The encoder supply is monitored for 8.2 V.

### Encoder Signals A0 to A7, B0 to B7

You can connect four different types of encoder:

- NAMUR encoders pursuant to DIN 19234 (with diagnostics):

  The signals are wired to terminals A0 to A7.
- 24 V incremental encoder:

  The signals A0/B0 to A7/B7 are connected via the terminals so labeled.
- 24 V pulse encoder with direction level:

  The count signals are wired to terminals A0 to A7. The directional signals are wired to terminals B0 to B7.
- 24 V initiators:

  The signals are connected to terminals A0 to A7.

  > **Note**
  >
  > An external 24 V DC voltage supply line must be used for the 24 V encoder supply.

### Digital Inputs I0 to I7 (Hardware Gates)

You can use the digital inputs I0 to I7 for the gate control of the counter.

One digital input is available for each count channel with which you can start and stop the corresponding counter.

The digital inputs are operated at a rated voltage of 24 V.

### Digital Outputs Q0 to Q7

The FM 350-2 can trigger control processes directly through digital outputs Q0 to Q7.

One digital output is available per counter.

The digital outputs are powered by the FM 350-2 24 V voltage supply.

The digital outputs are current-sourcing switches and support a load current of 0.5 A. These outputs are protected against overload and short circuit-proof.

> **Note**
>
> Relays and contactors can be connected directly without external circuitry.

#### Selecting channel settings (S7-300, S7-400)

##### Channel setting

This is the field in which you set the function of the module channels:

- Single counter: Each channel in the selected group operates as a separate counter.
- Dosing counter: 4 counter channels are combined to form one dosing channel. This channel can control up to 4 dosing units.

You can make the following selections:

- Channels 0 to 7 as single counters
- Channel 0 as dosingcounter, channels 4..7 as single counters
- Channels 0..3 as single counters, channel 4 as dosingcounter
- Channel 0 as dosing counter and channel 4 as dosing counter

#### Enabling counter input for NAMUR encoders (S7-300, S7-400)

##### NAMUR encoder

This is the field in which you define the counter inputs to which a NAMUR encoder is to be connected.

You can make the following selections:

- Counter input channel 0 to 3 NAMUR

  and/or
- Counter input channel 4 to 7 NAMUR

See also: "[NAMUR signals](#namur-signals-s7-300-s7-400)" and "[Enabling hardware monitoring for NAMUR encoders](#enabling-hardware-monitoring-for-namur-encoders-s7-300-s7-400)"

#### NAMUR Signals (S7-300, S7-400)

##### NAMUR encoder

The encoder supplies one count signal in accordance with DIN 19234 and this must be connected to terminal A0 to A7 of the front connector.

You can wire a directional signal (24 V) to terminal B0 to B7 of the relevant counter. If your encoder does not return a corresponding signal, you can also generate and interconnect a relevant identifier within the S7 system, or use a suitable process signal.

The diagram below shows the time profile of a NAMUR encoder with directional level.

You can enable inputs A0 to A3, A4 to 7, or all inputs for NAMUR encoder connections.

##### How are the Signals Monitored?

You can assign parameters for the FM 350‑2 so the module monitors A0 to A7 for wire break and short circuit. This also indirectly results in the 8.2 V encoder supply being monitored. This is only possible with NAMUR encoders.

If a wire break or short circuit is detected, the group error LED lights up, you can also assign a diagnostic interrupt to be triggered.

#### Mapping the count and measurement values of all channels in the expanded user data (S7-300, S7-400)

##### Mapping of all count and measurement values in the user data image

Under certain conditions, all count and measurement values can be mapped in the user data image (use of expanded user data).

The following basic conditions apply to use of expanded user data:

- Configure the module "6ES7350-2AH01-0AE0 E" from the module catalog.
- The check box "Use extended user data interface" must be selected.
- The extended user data can only be used on a distributed basis behind an IM 153‑2 V5.0 (article number 6ES7153‑2BA01‑0XB0 V5.0) or higher.
- 40 bytes of input data are now available.
- The update cycle of the extended user data differs from the update cycle of the first 16 bytes of input data. The update cycle of the extended user data is approximately 100 to 200 ms depending on the PROFIBUS DP cycle time.

The structure of the first 16 bytes in the user data is the same as before in each operating mode. The additional 24 bytes are defined by the operating mode.

Depending on the operating mode, the values (count or measurement value) of channels 2 to 7 are stored in the expanded user data. These values are preassigned to "0". The input data are occupied with 0 as long as valid values have not been detected.

Structure of the extended user data as defined by the operating mode:

| Byte | Counting modes | Measuring modes | Dosing mode  (two dosing counters) |
| --- | --- | --- | --- |
| 0..15 | Same as previous – structure can be influenced via parameter assignment | Same as previous – structure can be influenced via parameter assignment | Same as previous – structure can be influenced via parameter assignment |
| 16..19 | Count value channel 2 | Measurement value channel 2 | 0 |
| 20..23 | Count value channel 3 | Measurement value channel 3 | 0 |
| 24..27 | Count value channel 4 | Measurement value channel 4 | Dosing counter channel 4 |
| 28..31 | Count value channel 5 | Measurement value channel 5 | 0 |
| 32..35 | Count value channel 6 | Measurement value channel 6 | 0 |
| 36..39 | Count value channel 7 | Measurement value channel 7 | 0 |

#### Specifying the high count limit (S7-300, S7-400)

##### Description

By configuring an high count limit together with a main count direction (up or down), you can reduce the maximum counting range. The parameterized counting range is then between 0 and the high count limit parameterized.

The high count limit can be any value between > 1 and the high count range limit.

#### Specifying measuring time (S7-300, S7-400)

##### Description

If you have selected a measuring mode, the module will update the measured values on a cyclic basis. The integration time is the product of the measuring time x 10ms. You set the measuring time as the factor n.

Detailed descriptions of the value ranges for the measuring time can be found under "[Measuring basics](Counting%20and%20measuring%20%28S7-300%2C%20S7-400%29.md#measuring-basics-s7-300-s7-400)".

#### Using a hardware gate (S7-300, S7-400)

##### Using a hardware gate

You can decide when or not the channel is to count with a hardware gate.

If counting is to be done with a hardware gate, check the check box "Use hardware gate".

#### Selecting gate function (S7-300, S7-400)

##### Description

When configuring the hardware and software gates, you can specify whether the internal gate is to cancel or interrupt counting.

- [Cancel counting](Counting%20and%20measuring%20%28S7-300%2C%20S7-400%29.md#cancel-and-interrupt-function-of-the-gate-s7-300-s7-400): The counting process restarts from the beginning after gate stop and restart.
- [Interrupt counting](Counting%20and%20measuring%20%28S7-300%2C%20S7-400%29.md#cancel-and-interrupt-function-of-the-gate-s7-300-s7-400): The counting process starts from the last counter value after gate stop and restart.

#### Selecting main count direction (S7-300, S7-400)

##### Description

By configuring a [main count direction](Counting%20and%20measuring%20%28S7-300%2C%20S7-400%29.md#main-count-direction-s7-300-s7-400) (up or down), you can limit the maximum counting range by setting an high count limit. The parameterized counting range is then between 0 and the high count limit parameterized.

A main count direction can only be selected in the "Count once", " Count periodically" and "Dosing" modes.

You can select the following behavior:

- Up
- Down

#### Setting hysteresis (S7-300, S7-400)

##### Function principle of the hysteresis

An encoder can stop at a certain position and then "oscillate" around this position. This state causes the counter level to fluctuate around a specific value. If there is a comparison value within this fluctuation range, for example, the associated output is switched on and off in accordance with the rhythm of the fluctuations. The FM 350‑2 has configurable hysteresis to prevent this switching in response to small fluctuations.

Values between 0 and 255 can be set for hysteresis.

The effect of hysteresis is as follows in accordance with the value parameterized:

| Hysteresis | Effect |
| --- | --- |
| Hysteresis value n = 0, 1 | The hysteresis has no effect (switched off)  The output reacts to the slightest change in the counter value. |
| 2 ≤ Hysteresis value n ≤ 255 | The hysteresis takes effect.  The output does not react until the counter value is offset by n units to the comparison value. |

The hysteresis applies to both high limit violated and low limit violated.

#### Comparison values and outputs (S7-300, S7-400)

##### Description

You can store eight comparison values on the FM 350‑2, each for one of the eight digital outputs (comparison value 0: Q0, comparison value 1: Q1, etc.). The relevant output can be set depending on the counter level and the comparison value.

You can assign four comparison values for each dosing channel of the FM 350‑2 in "[Dosing](Counting%20and%20measuring%20%28S7-300%2C%20S7-400%29.md#dosing-only-for-fm-350-2-s7-300-s7-400)" mode.

##### Comparison values

The comparison values are entered during parameter assignment.

When the CPU is in RUN mode you can enter the comparison values in the data block of the CNT2_WR / CNT2WRPN instruction (CMP_VAL0…7) and transfer them to the FM 350‑2 using jobs 30 to 37 or 60 to 62. The count is not affected by this.

The comparison values must lie within the count range of the respective operating mode; the count range limits cannot be used.

If you want to set the comparison value to the initial or final value in an operating mode with a main count direction, the digital outputs will behave as follows:

| Parameterized behavior of digital outputs | Response of assigned digital outputs |
| --- | --- |
| On when counter value >= comparison value | The output is always switched on irrespective of the counter value |
| On when counter value <= comparison value | The output is always switched off irrespective of the counter value |

##### Enabling the Outputs

Before the outputs can be set, you must first enable them by setting the relevant bits in the counter DB. If you reset one of these bits, the associated output will be switched off immediately. The bits are transferred between the data block and the module by the CNT2_CTR instruction.

| Output | ...is enabled by |
| --- | --- |
| Q0…7 | CTRL_DQ0…7 |

##### Output switch-off

The outputs are switched off by the following events regardless of the parameter assignment:

- Module watchdog triggered (internal error)
- Removal of the enable bits (CTRL_DQ0...7 for Q0...7 in the DB)

##### Controlling the outputs

Irrespective of output switching via the comparators, you can set and reset the outputs when they are enabled by CTRL_DQ0…7 using the SET_DQ0…7 control signals.

The following applies to the relationship between controlling and switching: Controlling has priority over switching using the comparators. This means:

- The output can be used as a digital output if switching using the comparator is not set for the output.
- You can continue to control the outputs using SET_DQ0…7 if you have selected a compare function for the outputs. You can use this setup in the user program to simulate the effect of the compare functions:

  - The output is set with the positive edge of SET_DQ0…7.
  - A SET_DQ0…7 negative edge resets the output.

  Note that the comparators are still active and can set or reset the output if the comparison result changes.

  > **Note**
  >
  > An output set with SET_DQ0…7 cannot be reset with the comparator.

#### Enabling hardware interrupt trigger (S7-300, S7-400)

##### Introduction

The FM 350‑2 allows you to set which events are to trigger a hardware interrupt during operation.

##### What is a hardware interrupt?

If you want to set a response to a specific event independently of the CPU cycle, the FM 350‑2 can initiate a hardware interrupt. The CPU then interrupts the cyclical program and processes the hardware interrupt OB.

##### Enabling the hardware interrupt

Hardware interrupts are enabled in two steps:

1. When configuring the hardware, you enable the interrupts for the module in the basic parameters and choose whether the module is to trigger diagnostic and/or hardware interrupts.
2. You then select the relevant events under "Enable hardware interrupts":

   - [Events to trigger a hardware interrupt in counting modes](#events-to-trigger-a-hardware-interrupt-in-counting-modes-s7-300-s7-400)
   - [Events to trigger a hardware interrupt in measuring modes](#events-to-trigger-a-hardware-interrupt-in-measuring-modes-s7-300-s7-400)

##### Hardware Interrupt OB, e.g. OB40

If a hardware interrupt occurs, the user program is interrupted, the data is transferred from the module to the [start information of OB40](#ob-40-start-information-s7-300-s7-400) and OB40 is called. The hardware interrupt is acknowledged by exiting OB 40.

If no hardware interrupt has been programmed, the CPU goes into STOP mode. If you then switch back to RUN, the hardware interrupt requirements will be deleted.

##### Lost hardware interrupt

If an event occurs which is supposed to trigger a hardware interrupt but there is an identical, previous event which has not yet been acknowledged, no further hardware interrupt will be triggered; the hardware interrupt will be lost. This may lead to the "Hardware interrupt lost" diagnostic interrupt depending on parameter assignment.

If there is less that 2 ms between two events which should trigger a hardware interrupt, the second hardware interrupt will be lost without a diagnostic interrupt being triggered.

##### Default setting

No hardware interrupts are enabled in the default setting.

#### Events to trigger a hardware interrupt in counting modes (S7-300, S7-400)

##### Which events can trigger a hardware interrupt in counting modes?

The following events can trigger a hardware interrupt in counting modes during operation of the FM 350‑2:

- The hardware gate opening
- The hardware gate closing
- High and low limit violation
- Reaching a comparison value

  Enter the comparison value in the text box. The comparison values must lie within the count range of the respective operating mode; the count range limits cannot be used.

Select the check box for the relevant events.

You can select a number of different events to trigger a hardware interrupt depending on the operating mode set. You must observe the relevant basic conditions for hardware interrupts on reaching a comparison value (for "[On when counter value >= comparison value](#on-when-counter-value-comparison-value-s7-300-s7-400)" and "[On when counter value <= comparison value](#on-when-counter-value-comparison-value-s7-300-s7-400-1)").

#### Triggering diagnostics interrupts (S7-300, S7-400)

##### What is a diagnostic interrupt?

You can determine reactions to internal or external errors in the user program by configuring a diagnostic interrupt which interrupts the CPU cyclic program and calls the diagnostic interrupt OB 82.

##### What events can trigger a diagnostic interrupt?

The following events can trigger a diagnostic interrupt:

- Channel error in parameterized channel
- No module parameter assignment
- Error in module parameter assignment
- Watchdog reacts
- Hardware interrupt lost
- Signal line monitoring NAMUR encoder reports error
- Fault in 8.2 V NAMUR encoder supply

##### Enabling a diagnostic interrupt

When configuring the hardware, you enable the interrupts for the module in the basic parameters and choose whether the module is to trigger diagnostic and/or hardware interrupts.

##### Responses to a diagnostic interrupt

When an event occurs that could trigger a diagnostic interrupt:

- The diagnostics information is stored in the [diagnostic data records DS0 and DS1](#diagnostic-data-record-ds0-and-ds1-s7-300-s7-400) in the module.
- The group error LED lights up.

  The group error LED goes out when the error is eliminated.
- The diagnostic interrupt OB (OB 82) is called.
- The diagnostic data record DS0 is entered in the start information of the diagnostic interrupt OB.
- The count continues unchanged if an OB 82 has been programmed. If no OB 82 has been programmed, the CPU will go into STOP mode.

##### Default setting

No diagnostic interrupts are enabled in the default setting.

#### Diagnostic data record DS0 and DS1 (S7-300, S7-400)

##### Diagnostic data record DS0 and DS1

The information about which event triggered a diagnostic interrupt is stored in the diagnostic data records DS0 and DS1. The diagnostic data record DS0 comprises four bytes. DS1 comprises 16 bytes; the first four bytes are identical to DS0.

##### Reading a data record from the module

[Diagnostic data record DS0](#assignment-of-diagnostic-data-record-ds0-s7-300-s7-400) is automatically transferred to the start information when the diagnostic OB is called. These four bytes are written to the local date (bytes 8 to 11) of OB 82.

You can read out the [diagnostic data record DS1](#assignment-of-diagnostic-data-record-ds1-s7-300-s7-400) (and thus also the content of the DS0) from the module with the instruction DIAG_RD. This only makes sense if an error is reported in a channel in the DS0.
