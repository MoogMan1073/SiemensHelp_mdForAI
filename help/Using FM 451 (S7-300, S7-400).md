#### Auxiliary voltage for the encoder supply (1L+, M)

Here, you connect a 24 V DC auxiliary voltage for the encoder supply. The reference potential of this supply (M) is connected to the ground of the load current supply (M) in the FM 451.

The 24 V DC auxiliary voltage for the encoder supply is monitored for undervoltage and wire break.

This supplies 24 V DC and 5.2 V DC at the encoder interface (sub D sockets encoder CH1, CH2 and CH3) to the various types of encoders.

| Symbol | Meaning |
| --- | --- |
|  | **Caution** |
| Ensure the correct polarity of the 24 V DC auxiliary voltage for the encoder supply (1L+, M).  If you reverse the polarity of the 24 V DC auxiliary voltage for the encoder supply, the module will be damaged and will have to be replaced. |  |

#### Auxiliary voltage for the load current supply (2L+, 3L+, 4L+, M)

You must connect the 24 V DC auxiliary voltages for the load current supply of the digital outputs to the 2L+, 3L+, 4L+, and M terminals.

| Symbol | Meaning |
| --- | --- |
|  | **Caution** |
| Verify the correct polarity of the 24 V DC auxiliary voltage for the load current supply (2L+, 3L+, 4L+, M).  If you reverse the polarity of the 24 V DC auxiliary voltage for the load current supply, the module will be damaged and will have to be replaced. |  |

#### Wiring information for 24 V DC

When wiring, note that the terminals 1L+ to 4L+ must be interconnected so that the module will operate error-free.

Start by wiring the 24 V DC to terminal 38 and connect the auxiliary voltage of terminal 37 to terminal 26. Use the same procedure for the other terminals.

Alternatively, you can also connect separate voltage supplies to the 1L+ (encoder), 2L+ (channel 1), 3L+ (channel 2), and 4L+ (channel 3) connections. Make sure that all voltage supplies have a common ground potential. In this case, synchronization of the axes will be retained following a power failure in the auxiliary voltages for the load current supply.

#### Load current supplies

The DC load current supply must meet the following requirements:

Only a safe, isolated extra-low voltage of ≤ 60 V DC may be used as the load current supply. Safe isolation can be implemented, for example, by adhering to the specifications

- VDE 0100 Part 410 / HD 384-4-41 / IEC 364-4-41  
   (as protective extra low voltage) and/or
- VDE 0805 / EN 60950 / IEC 950  
  (as safety low voltage SELV) or
- VDE 0106 Part 101.

#### 12 digital inputs (1I0 to 3I3)

The FM 451 has 4 digital inputs per channel.

You can connect bounce-free switches (24 V current sourcing) or non-contact sensors (2- or 3-wire proximity switches) to the 12 digital inputs.

The digital inputs are not monitored for short circuits or wire break and have a non-isolated connection to the encoder supply ground.

A separate LED indicates the state of each input.

#### 12 digital outputs (1Q0 to 3Q3)

The FM 451 has 4 digital outputs per channel.

The power unit is controlled via the digital outputs. The function of the digital outputs depends on the [control mode](#overview-of-control-modes-s7-300-s7-400).

The digital outputs are not monitored for short circuits or wire break and have a non-isolated connection to the encoder supply ground.

The associated LED indicates the state of each output.

### Writing and reading machine data (S7-300, S7-400)

#### Writing, enabling and reading machine data

You can use the machine data to adapt the positioning module to the axis and the encoder.

You can change and read parameters during operation using the user program.

All parameters are stored in the parameter DB:

- Machine data is located in the parameter DB at the addresses 4.0 to 116.0.

Enter the parameter DB number in the corresponding channel DB.

You can enter the parameters with the DB Editor or, simply and easily, enter them in the "Drive", "Axis", "Encoder" and "Increments" dialogs before writing them to the parameter DB with the "Export" function.

You can use the "Import" function to import parameters from an existing parameter DB to the dialogs.

#### Initial parameter assignment

If the channel does not yet contain any machine data, proceed as follows for initial parameter assignment without the parameter assignment dialogs:

1. Enter the new values in the parameter DB and then save them.
2. Download the parameter DB to the CPU.
3. Set the following trigger bit in the channel DB for the job:

   - Write machine data (MDWR_EN = 1).
4. Call the [ABS_CTRL](FM%20x51%20Positioning%20%28S7-300%2C%20S7-400%29.md#abs_ctrl-s7-300-s7-400) or [ABS_CTRL_451](FM%20x51%20Positioning%20%28S7-300%2C%20S7-400%29.md#abs_ctrl_451-s7-300-s7-400) instruction in the cyclic user program.

#### Modifying machine data

Proceed as follows to change machine data using the user program:

1. Enter the new values in the parameter DB.
2. Set the trigger bits in the channel DB for the jobs:

   - Write machine data (MDWR_EN = 1)
   - Activate machine data (MD_EN = 1)
3. Call the [ABS_CTRL](FM%20x51%20Positioning%20%28S7-300%2C%20S7-400%29.md#abs_ctrl-s7-300-s7-400) or [ABS_CTRL_451](FM%20x51%20Positioning%20%28S7-300%2C%20S7-400%29.md#abs_ctrl_451-s7-300-s7-400) instruction in the cyclic user program.

If you set the trigger bits for these jobs all at once, the ABS_CTRL instruction (or ABS_CTRL_451) makes sure the jobs are processed in the correct order.

Otherwise change the machine data always in the following sequence:

- Write machine data
- Enable machine data

#### Reading machine data

In order to read the current machine data from a channel, proceed as follows:

1. Set the following trigger bit in the channel DB:

   - Reading machine data (MDRD_EN = 1)
2. Call the [ABS_CTRL](FM%20x51%20Positioning%20%28S7-300%2C%20S7-400%29.md#abs_ctrl-s7-300-s7-400) or [ABS_CTRL_451](FM%20x51%20Positioning%20%28S7-300%2C%20S7-400%29.md#abs_ctrl_451-s7-300-s7-400) instruction in the cyclic user program.

The current machine data is then stored in the parameter DB on the CPU.

| 35.0 | MDWR_EN | BOOL | FALSE | 1 = Write machine data |
| 35.1 | MD_EN | BOOL | FALSE | 1 = Enable machine data |

### End of a positioning operation (S7-300, S7-400)

#### Definition

The end of a positioning operation is displayed by the checkback signal WORKING = 0. It can be reached in three different ways:

- Target approach
- End
- Cancel

#### Monitoring

The following monitoring functions are active towards the end of a positioning operation:

- Monitoring time

  The monitoring time is retriggered at the switch-off point for the last time and then loses its validity with the end of positioning.

  The end of the positioning operation must be reached within this time. If it is not, the outputs will be switched off and the operating error "Error during target approach" (error number 5) issued.
- Monitoring the target range

  The FM x51 sets a symmetrical range around each target, thereby defining the positioning precision of your application. The axis must come to a standstill within this range during a target approach. A value 0 switches off the tolerance during the target approach.
- Monitoring the standstill velocity

  The standstill velocity is used determine whether the drive comes to a standstill within the target range. The FM x51 checks whether the actual drive speed is less than the stationary drive speed when the switch-off point has been reached.

  It must be less than the standstill velocity within the target range. The FM x51 will otherwise report the operating error "Target range overrun" (error number 10).

  Falling short of the standstill velocity is monitored only once per target approach.

  Note that the velocity can briefly fall below the standstill velocity for the velocity determination of the module if the axis moves at a very slow positioning velocity (less than 2 pulses per 8 ms).
- Monitoring the stationary range

  Once a positioning operation ends, the drive is monitored to see whether it remains at an approached target position or drifts off.

  The standstill range is monitored

  - after the FM x51 has reported the checkback signal "PR"
  - if the monitoring time is exceeded
  - if the velocity falls below the standstill velocity.

  If the standstill range is exited without a valid traversing job, the FM x51 reports the "Stationary range exited" error (error number 6).

#### Target approach

The target approach in the "Absolute incremental approach" and "Incremental approach" modes starts when the switch-off point has been reached. As of this point the drive is switched off and the FM x51 adopts the monitoring functions.

Depending on the monitoring you have parameterized, there are a number of different scenarios in which the checkback signal "PR (POS_RCD)" is generated. The positioning operation is cancelled if no "PR (POS_RCD)" checkback signal is generated.

**Scenario 1:** You have assigned parameters as follows:

- Target range (TRG_RANGE) > 0
- Standstill velocity (ZSPEED_L) > 0
- Monitoring time (MON_TIME) > 0

PR is generated when the velocity falls below the standstill velocity and the target range has been reached. It does not matter which condition is fulfilled first.

PR is not generated if the actual value has not reached the target range within the monitoring time or the target range has been overrun without the velocity falling below the standstill velocity.

### "Jog" mode (S7-300, S7-400)

#### Definition

In "Jog" mode, you can move the drive in one direction by pushing a button. You must install a button for both directions (plus and minus) respectively. You can use the "Jog" mode both for a synchronized as well as for an unsynchronized axis.

In the case of an unsynchronized axis, jogging is a positioning in the specified direction.

For a synchronized axis (linear axis), jogging involves a positioning to the software limit switches.

#### Requirement

The axis parameters must be assigned.

#### Sequence of events for "Jog" mode

### "Incremental approach" mode (S7-300, S7-400)

#### Definition

With the "Incremental approach" mode, the FM 451 can move the drive.

- toward **absolute** targets,
- **relatively** by an incremental distance in a specified direction.

You specify the target or the relative distances as increment dimensions of the FM 451. You have the possibility of entering up to 100 incremental dimensions in a table that are valid both for the "Relative incremental approach" mode as well as for the "Absolute incremental approach" mode. Independently of the incremental dimension table, you can specify the distance using [increment dimension 254](#increment-number-254-s7-300-s7-400) and [incremental dimension 255](#increment-number-255-s7-300-s7-400)

The [incremental dimension 252](#increment-number-252-s7-300-s7-400) gives you the option of moving the drive initially in a specified direction without a target (continuous traversing). When an external event occurs (xI3), the programmed target is then positioned upon.

#### Requirements

- The axis parameters must be assigned.
- The axis must be synchronized.
- The increments must be available on the module.

#### Interpretation of the increments

The FM 451 interprets the specifications differently, depending on the incremental approach you select.

- Absolute incremental approach: The incremental dimensions are interpreted as absolute target position.  
   With a rotary axis, the end of the axis may not be passed.
- Relative incremental approach: The incremental dimensions are interpreted as distance difference from the start position.  
   With a rotary axis, the end of the axis may not be passed (multiple revolutions are permitted).

> **Note**
>
> Only positive increments are permissible for the "Relative incremental approach" mode.
>
> The incremental dimensions result from the DIR_P or DIR_M direction specifications.

#### Distance-to-go

The distance-to-go is the difference between the target (increment) and process value. The distance-to-go is

- positive, if the target has not yet been reached
- negative, if the target has already been passed.

In the case of a rotary axis, the displayed distance-to-go cannot be used.

#### Shutting down the Incremental approach

The "Incremental approach" mode is shut down when the FM 451 receives a STOP signal (STOP = 1).

After traversing is shut down, the distance-to-go remains.

The distance-to-go for "Relative incremental approach" and "Absolute incremental approach with incremental dimension number 252" can be traversed to the end if

- the mode is unchanged, and
- the increment number is unchanged, and
- the direction is unchanged, and
- the remaining distance-to-go is greater than the assigned switch-off difference.

You traverse the residual distance by starting "Relative incremental approach" or "Absolute incremental approach with incremental dimension number 252" again without making any changes.

#### Canceling the incremental approach

The "Incremental approach" mode is cancelled when the "Drive enable" signal is deleted (DRV_EN = 0).

With the job "Delete distance-to-go" you can delete any awaiting distance-to-go.

With "Incremental approach with incremental dimension number 252", traversing restarts with continuous traversing after "Delete residual distance".

You also delete the awaiting distance-to-go by calling a different mode or by starting the mode in the other direction.

### The "Set actual value" / Set actual value on-the-fly / "Undo Set Actual Value" job (S7-300, S7-400)

#### Definition

You use the "Set actual value" or "Set actual value on-the-fly" jobs in order to assign a new coordinate to the current encoder value. The operating range is projected on a different area of the axis.

You determine the shift in the operating range with (**ACT**<sub>new</sub>**-** **ACT**<sub>current</sub>).

- **ACTUAL**
  <sub>new</sub> is the default value
- **ACTUAL**
  <sub>current</sub> is the actual value at the time of the execution

#### Requirements

- The axis parameters must be assigned.
- The axis must be synchronized.

For "Set actual value", the positioning process must be finished.

#### Job sequence

1. Enter the coordinate AVAL or FVAL for the actual value (**ACT**<sub>new</sub>).

   - Linear axis:

     You must select the specified actual value so that the software limit switches are still within the permissible traversing range after the job is called.

     Rule for the defined actual value:

     0 ≤ actual value < end of rotary axis

- The CPU program requires an interval of sufficient length between the off and on edges at input xI3 in order to correctly evaluate the result of the measurement before a new measurement starts.

### "Loop traverse" function switch (S7-300, S7-400)

#### Definition

With "loop approach" you specify the direction in which a target is approached with a frictional connection. You can use loop approach if a frictional connection between the motor and axis can only be ensured in one direction.

A target that is approached in the direction opposite the specified direction is initially overrun. Then, the module performs a direction reversal and approaches the target in the specified direction.

#### Requirements

- The axis parameters must be assigned.
- The axis must be synchronized.
- In the case of a loop approach in the opposite direction to that of the traversing direction of the target the maximum target position is:

  - traversing direction plus

    **Target** < **SLE - ½ Target range - Switch-off difference plus - Changeover difference minus**
  - traversing direction minus

    **Target** < **SLE - ½ Target range + Switch-off difference minus + Changeover difference minus**
- A loop approach will not be carried out if the target is being approached in the direction of the loop approach. In this case, an incremental approach without an about-turn in direction will be carried out.
- The sequence of the ["incremental approach"](#incremental-approach-mode-s7-300-s7-400) operating mode must be known.

#### Sequence of events for the loop approach

1. Set the control signal for the "absolute/relative incremental approach" operating mode (MODE_IN = 5/4).
2. Set the control signal for the drive enable (DRV_EN = 1).
3. Set the "Enable input" (EI_OFF = 1) function switch or wire the enable input for the corresponding channel.
4. Enter the incremental dimension number (MODE_TYPE = 1 … 100, 254, 255).
5. Set the "Loop traverse" function switch (PLOOP_ON / MLOOP_ON = 1).
6. Start the incremental approach.

### "Enable input" function switch (S7-300, S7-400)

#### Definition

The enable input is an external input with which positioning can be enabled by an external event.

#### Evaluate enable input (EI_OFF = 0)

You have the following options for evaluating the enable inputs:

- **edge-controlled (**
  **EI_TYPE = 1**

#### Do not evaluate enable input (EI_OFF = 1)

If you disable the evaluation of the enable input, an operating mode starts immediately after the start signal is detected. In this case, you cannot prepare an operating mode and start it at a future specified time.

### Querying checkback signals for diagnostics (S7-300, S7-400)

#### Definition

The "checkback signals for the diagnostics" informs you of occurring diagnostic events.

#### Sequence

2. If it is not possible to call an operating mode or control an active operating mode, or if this has been carried out incorrectly, the module sets an OT_ERR operator error in the feedback interface. The cause of error is then entered in the diagnostic buffer. While the operator error exists, you can neither start a new operating mode nor continue with the halted one. 3. The module sets the DATA_ERR bit in the feedback interface when it detects incorrect data in a write job. The cause of error is then entered in the diagnostic buffer.
4. The checkback signals are stored in the channel DB.
5. When the diagnostic buffer is read, the module sets the DIAG bit back to 0 in all channels.

#### Specifying changeover difference (S7-300, S7-400)

##### Changeover difference

The changeover difference defines the distance from the destination at which the drive is switched over from rapid feed to creep feed.

The specified values apply to all targets approached by the positioning module.. An incremental approach with the incremental dimension 255 is an exception. A special changeover difference is specified for this.

##### Rules for changeover difference

- The changeover difference plus and the changeover difference minus do not have to be arranged symmetrically around the target. The values can differ.
- The changeover difference must be greater than the switch-off difference.
- The changeover difference must lie within the operating range.
- The changeover difference must be less than the end of rotary axis.
- Select a sufficient distance between the changeover point and the switch-off point to ensure that the drive can safely switch from rapid traverse to creep speed.
- The changeover point and the switch-off point must be separated from each other by a time interval of at least 8 ms.

##### Specifying standstill range (S7-300, S7-400)

###### Standstill range

The standstill range lies symmetrically around the respective target. In this range, monitoring is performed to determine whether the drive remains on an approached target position or drifts away from it. The monitoring begins when the positioning module has reported "Position reached".

If the standstill range is exited without a valid traversing job, the positioning module signals an error.

###### Rules for standstill range

- The standstill range lies symmetrically around the respective target.

  (Target – ½ × standstill range) ≤ target ≤ (target + ½ × standstill range)
- The standstill range must be greater than or equal to the target range.
- The coordinate values for the standstill range must lie in the operating range.
- A setting of "0" deactivates the standstill monitoring function.

##### Specifying software limit switch (S7-300, S7-400)

###### Software limit switch

Software limit switches only exist for the linear axis.

The operating range is the range limited by the software limit switches.

The software limit switch start must always be smaller than the software limit switch end.

The software limit switches are monitored after the axis is synchronized.

- When using an incremental encoder:

  The synchronization of the axis is achieve by a reference point approach or by setting a reference point.
- When using an absolute encoder (SSI):

  The axis is synchronized when the module has received a complete, error-free frame for the associated channel.

  The absolute encoder must cover at least the operating range including the software limit switches.

###### Relationship between the operating range, encoder range and traversing range

- The operating range is the range which you specify for your job using the software limit switches.
- The encoder range represents the range explicitly covered by the encoder. For a linear axis, the module assigns the encoder range symmetrically across across the operating range. In other words, the module shifts the encoder range in order to equalize the distances between the software limit switches and the ends of the encoder range (see figure below).
- This traversing range represents the value range the positioning module can process. The traversing range is dependent on the resolution.

The following figure shows the relationship between software limit switches, the operating range, the encoder range and the traversing range.

##### Specifying end of rotary axis (S7-300, S7-400)

###### End of rotary axis

The value "end of rotary axis" is the theoretical maximum value that the actual value can reach. The theoretically highest value is however never displayed, because it physically identifies the same position as the start of the rotary axis (0).

The maximum value that will be displayed for a rotary axis has the value:

End of rotary axis [µm] - resolution [µm / pulse] x 1 [pulse]

**Example:** End of rotary axis 1000 mm, resolution 1000 µm / pulse

The display jumps:

- with a positive rotational direction from 999 mm to 0 mm
- with a negative rotational direction from 0 mm to 999 mm.

###### Rotary axis with absolute encoders

The rotary axis range (0 to end of rotary axis) of a rotary axis with an absolute encoder must correspond exactly to the encoder range of the absolute encoder:  
End of the rotary axis [µm] = number of encoder revolutions x distance/encoder revolutions [µm]

###### Absolute encoder adjustment procedure (S7-300, S7-400)

###### Determining the correct absolute encoder adjustment

After initial parameter assignment, further steps must be taken to ensure that a correct relationship is created between the encoder and coordinate system. The procedure shown in the next section is based on the use of the parameter assignment dialogs.

1. Traverse the axis to a defined, reproducible point to which a unique coordinate is assigned.

   This can, for example, be the "software limit switch end".
2. Call the job "[Set reference point](#set-reference-point-job-s7-300-s7-400)" with the coordinate of the point defined under 1.

   The positioning module now determines an encoder value for the reference point coordinate entered in the DB channel (REFPT in channel DB), namely the absolute encoder adjustment. You can read this value with the "[Read encoder data](#read-encoder-data-job-s7-300-s7-400)" job.
3. Enter the value read in the "absolute encoder adjustment" field.
4. Save your parameter assignment to the corresponding parameter DB using the export function.
5. Download the data in the HWCN to the CPU.
6. Restart the CPU (cold restart) to apply the data.

> **Note**
>
> You perform this adjustment just once during the commissioning. After a configuration, the position module is synchronized during start-up as soon as a complete, faultless message frame of the encoder is received after start-up.

###### Otherwise: Mechanical adjustment of the encoder

A correct relationship between the coordinate system and the encoder can be achieved as follows:

1. Move the axis to a defined, reproducible point to which a unique coordinate is assigned.

   This can, for example, be the "software limit switch start".
2. Enter this coordinate value in the machine data as "reference point coordinate".
3. Read the encoder value displayed at this position with the "[Read encoder data](#read-encoder-data-job-s7-300-s7-400)" job.
4. Enter this value in the machine data as the "absolute encoder adjustment".

After the parameter assignment a correct actual value will always be shown.

Instead of steps 3 and 4, you can also set the encoder to zero using "Reset", if available. Then enter the value "0" in the machine data as the absolute encoder adjustment.

#### Selecting monitoring (S7-300, S7-400)

##### Monitoring

**Wire break**

When monitoring is activated, the positioning module monitors all cables for a 5 V incremental encoder and an absolute encoder. The monitoring detects:

- Wire break
- Short circuit on individual cables
- Edge distance of counting pulses (also for 24 V incremental encoder)

For monitoring for a 24 V incremental encoder, you must set [MON_TIME > 0 as the monitoring time](#specifying-monitoring-time-s7-300-s7-400).

**Frame error**

The module monitors the message frame of an absolute encoder (SSI) for:

- Start and stop bit errors

You cannot disable the message frame error monitoring with absolute encoders (SSI).

**Missing pulses (incremental encoder)**

An incremental encoder must always return the same number of increments between two successive zero marks.

The positioning module checks to see if the zero mark of an incremental encoder is occurring at the correct encoder state.

For encoders without zero marks you must switch off the monitoring for missing pulses.

You must also disable the wire-break monitoring or interconnect the N and /N zero mark inputs externally.

### Control and feedback interface of the FM 451 (S7-300, S7-400)

#### Application

In special applications or in an alarm level, a particularly fast access to checkback and control signals could be required. You can reach this data directly via the input and output areas of the module.

To coordinate startup following each module startup, for example, after plugging a module or after a STOP-RUN transition of the CPU, you must call the [ABS_CTRL](FM%20x51%20Positioning%20%28S7-300%2C%20S7-400%29.md#abs_ctrl-s7-300-s7-400) instruction until the end of the startup is indicated by RETVAL = 0. Then, do not use the ABS_CTRL anymore.

> **Note**
>
> It is not possible to use the ABS_CTRL together with a write access.

#### Triggering diagnostic interrupts (S7-300, S7-400)

##### Processing interrupts

The positioning module can trigger a diagnostic interrupt. These interrupts are processed in an interrupt OB. If an interrupt is triggered without the corresponding OB being loaded, the CPU switches to STOP.

Enable the processing of the diagnostic interrupts as follows:

1. Select the module in HWCN.
2. Select "Edit" > Object Properties > Basic Parameters" to enable the diagnostic interrupt.
3. Save and compile the configuration.
4. Download the configuration to the CPU.

##### Overview of diagnostic interrupts

The following events and errors trigger a diagnostic interrupt:

- Operating error
- Machine data error
- Diagnostics errors

These errors are explained in detail in "[Error classes](#error-classes-s7-300-s7-400)".

##### Reaction of the positioning module to an error with diagnostic interrupt

- The positioning is canceled.
- The synchronization is deleted in the case of the following diagnostic interrupts:

  - Front connector missing, external auxiliary voltage for encoder supply missing
  - Zero mark error detected, cable fault (5 V encoder signals)
  - Traversing range exceeded (indicated by an operating error)
  - Set process value cannot be executed (indicated by an operating error).
- With one exception, control signals START, DIR_P und DIR_M are no longer processed.

  Exception:

  In the case of an operating error it is still possible to jog in the direction of the operating range.
- Function switches and jobs will no longer be processed.

##### Positioning module detects an error ("incoming")

A diagnostic interrupt is "incoming", if at least one error is given. If not all errors are remedied, the remaining existing errors will be reported again as "incoming".

Sequence:

1. The positioning module detects one or more errors and generates a diagnostic interrupt. Either the "INTF" or "EXTF" LED is illuminated. The error is then entered in the diagnostic buffer.
2. The CPU operating system calls the diagnostic interrupt OB, e.g. OB 82.
3. You can evaluate the start info of OB 82.
4. The OB82_MOD_ADDR parameter indicates which module triggered the interrupt.
5. For additional information, call the [ABS_DIAG](FM%20x51%20Positioning%20%28S7-300%2C%20S7-400%29.md#abs_diag-s7-300-s7-400) or [ABS_DIAG_451](FM%20x51%20Positioning%20%28S7-300%2C%20S7-400%29.md#abs_diag_451-s7-300-s7-400) instruction.

##### The positioning module detects the transition to the error-free state ("outgoing")

A diagnostic interrupt is then only "outgoing" if the last error on the module has been remedied.

Sequence:

1. The positioning module detects that all errors have been eliminated and triggers a diagnostic interrupt. The "INTF" or "EXTF" LED is no longer illuminated. The diagnostic buffer is not changed.
2. The CPU operating system calls the diagnostic interrupt OB, e.g. OB 82.
3. The OB82_MOD_ADDR parameter indicates which module triggered the interrupt.
4. Evaluate the OB82_MDL_DEFECT bit.

   If this bit is "0", there are no errors on the module. You can end your evaluation here.

##### Diagnostic interrupts in dependence on the CPU status

- Diagnostic interrupts are blocked by the positioning module when the CPU is in STOP mode.
- If not all of the pending errors are eliminated while the CPU is in STOP mode, the positioning module reports the remaining errors as "incoming" events again at the next CPU transition to RUN.
- If all pending errors are eliminated while the CPU is in STOP mode, the module does **not** report its error-free state with a diagnostic interrupt at the next CPU transition to RUN.

#### Error evaluation in the user program (S7-300, S7-400)

##### Error response in the user program

In the user program you can respond selectively to errors. The following methods can be used:

- The RET_VAL return values of the integrated operations:  
  This value is determined again each time the instruction is called.

  RET_VAL = -1 is a group display for a synchronous error in a job or when communicating with the module.
- An _ERR error bit belongs to each job. This is a group display for an error in the job or one of its predecessors in the job chain:  
   The error bit is set for a write job and its subsequent jobs when a data error is reported by the module or a communication error occurs.

  With reading jobs the error bit is set for the job in question when a communication error occurs.

  The [ABS_CTRL](FM%20x51%20Positioning%20%28S7-300%2C%20S7-400%29.md#abs_ctrl-s7-300-s7-400) or [ABS_CTRL_451](FM%20x51%20Positioning%20%28S7-300%2C%20S7-400%29.md#abs_ctrl_451-s7-300-s7-400) instruction resets the error bits once a job has been executed. These should however be cancelled by the user program in the case of an error assessment.
- The DATA_ERR checkback signal as a group display for an error that the module has detected during a write job. The signal will be established again with the next write job.
- The OT_ERR checkback signal (operator error) as a group display for an error that the module has detected when starting a traverse. The error must be acknowledged after the cause has been acknowledged with OT_ERR_A = 1.
- The DIAG checkback signal is set when the contents of the diagnostic buffer have changed. This signal can come along later than the signals DATA_ERR und OT_ERR.
- The communication error JOB_ERR includes the [error code](#list-of-job_err-messages-s7-300-s7-400) in the case of a communication problem between the instruction and the module. The value is determined again after a job has been processed and is stored in the channel DB for the [ABS_CTRL](FM%20x51%20Positioning%20%28S7-300%2C%20S7-400%29.md#abs_ctrl-s7-300-s7-400) or [ABS_CTRL_451](FM%20x51%20Positioning%20%28S7-300%2C%20S7-400%29.md#abs_ctrl_451-s7-300-s7-400) instruction in the channel DB and for the [ABS_DIAG](FM%20x51%20Positioning%20%28S7-300%2C%20S7-400%29.md#abs_diag-s7-300-s7-400) or [ABS_DIAG_451](FM%20x51%20Positioning%20%28S7-300%2C%20S7-400%29.md#abs_diag_451-s7-300-s7-400) instruction in the diagnostics DB.
- The [ABS_DIAG](FM%20x51%20Positioning%20%28S7-300%2C%20S7-400%29.md#abs_diag-s7-300-s7-400) or [ABS_DIAG_451](FM%20x51%20Positioning%20%28S7-300%2C%20S7-400%29.md#abs_diag_451-s7-300-s7-400) instruction for reading the diagnostic buffer of the module. #### Diagnostics buffer of the module (S7-300, S7-400)

##### Diagnostic events

The module´s diagnostic buffer includes maximum 9 diagnostic events and is organized as a ring buffer.

A diagnostic event is written in the buffer if an "incoming" (error) message is detected. This can be a message, a synchronous error (data error, operator error) or even an asynchronous error (operational error and diagnostics error). From a cause of error several entries can occur as follow-up errors. Outgoing messages do not create any entries in the diagnostic buffer.

For each diagnostic event the following is given:

- Status (always incoming)
- Internal error
- External error
- Error class
- Error number
- Channel number
- Increment number (in the case of increment table errors)

When a diagnostic event is written in the diagnostic buffer, the DIAG = 1 checkback signal is set in all assigned channels.

With the [ABS_DIAG](FM%20x51%20Positioning%20%28S7-300%2C%20S7-400%29.md#abs_diag-s7-300-s7-400) or [ABS_DIAG_451](FM%20x51%20Positioning%20%28S7-300%2C%20S7-400%29.md#abs_diag_451-s7-300-s7-400) instruction, the diagnostic buffer can be transferred as a whole into a data block (diagnostic DB) or displayed via the evaluation dialog of the configuration software. If the diagnostic buffer is read, the module sets the checkback signal DIAG = 0.

> **Note**
>
> If the diagnostic buffer is read simultaneously from the ABS_DIAG or ABS_DIAG_451 instruction and the error evaluation dialog, it is possible that a new diagnostic event is not detected by the program.

The following table shows the possible operating errors and their respective causes:

| 1 | **Software limit switch start overrun** |  | Yes |
| Cause | The actual value lies outside of the operating range. |  |  |
| 2 | **Software limit switch end overrun** |  | Yes |
| Cause | The actual value lies outside of the operating range. |  |  |
| 3 | **Traversing range start overrun** |  | Yes |
| Cause | Traversing range limit overrun (coordinates of the traversing range limits belong with the traversing range). |  |  |
