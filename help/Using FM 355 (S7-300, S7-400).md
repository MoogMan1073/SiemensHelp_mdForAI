### Configuring the hardware (S7-300, S7-400)

#### Procedure

Configuration assumes that you have set up a project in which you can save the parameter assignment. The following briefly explains the most important steps.

1. Start the TIA portal and open the device configuration.
2. Select one module rack.
3. Select one controller module from your module catalog.
4. Move the controller module to the appropriate place on the module rack.
5. From the configuration table, note the input address of the module, for example 272.

   The value that you read is displayed in decimal format.

### Module parameters (S7-300, S7-400)

#### Procedure

When assigning parameters you set the module parameters.

1. Double click in the device view under the "Parameter setting" module. Select the "Properties" tab in the inspector window.

   **Result:** The properties are displayed in the inspector window.
2. Set the basic settings of the module and the other parameters.
3. Set the further block settings in the project view.
4. Save the entered parameters with "Project > Save".
5. Translate the project with "Edit > Translate".
6. Transfer the parameter data with the CPU in STOP mode by selecting "Online > Load in device..."

   **Result:** The data is located in the CPU´s memory and will be directly transferred from there to the module.
7. Carry out a CPU start-up.

#### Filter functions (S7-300, S7-400)

##### Procedure

To set a filter function, proceed as follows:

1. Double click in the device view under the "Parameter setting" module.
2. Navigate in the project view to "Analog Input > Input (No.) > Filter functions".
3. Set the following parameter:

   - To filter according to a time constant, activate the "Signal filter" option.

     If the "Signal filter" option is selected, the filter time constant can be set.
   - Filter time constant

     Can only be set if the "Signal filter" option is activated.
   - To linearize the transmitter signal, activate the "Square root filter" option.
   - To linearize the transmitter signal characteristic, activate the "Polygon filter" option.
4. Save the entered parameters with "Project > Save".

##### Signal filter

Filter 1. order, whose time behavior is set through the time constants.

##### Square root filter

Serves for the linearization of the transmitter signals where the actual value exists as a physical size, which is in a quadratic connection with the measured process size.

##### Polygon filter

Serves for the linearization of transmitter characteristics.

The 13 optional supporting points can be selected in:

- mA for current input
- mV for voltage input

For the transfer of the input signal into a different physical unit through linear interpolation between the initial value (low value) and final value (high value).

#### Realizing fixed value or cascade controller (S7-300, S7-400)

##### Fixed value or cascade controller

For the fixed value or cascade controller structures, the effective actual value is identical with the actual value A.

##### Select controller type

To realize a fixed value or cascade controller, proceed as follows:

1. Double click in the device view under the "Parameter setting" module.
2. Navigate in the project view to "Controller > Channel (No.)".
3. Select the option "Fixed value or cascade controller" in the "Controller type" area.
4. Save the entered parameters with "Project > Save".

##### Prepare setpoint

To prepare the setpoint, proceed as follows:

1. Double click in the device view under the "Parameter setting" module.
2. Navigate in the project view to "Controller > Channel (No.) > Controller difference".
3. Set the corresponding parameters in the different areas.

   - Setpoint source

     Select the source of the setpoint.
   - Safety setpoint

     Set the safety setpoint here. Also set the reaction to a CPU-failure and the start-up behavior.
   - Setpoint ramp

     Set the start-up time.
   - Setpoint limit

     Select the high and low setpoint limits.
4. Save the entered parameters with "Project > Save".

##### Select signal source for actual value A

To select the signal source for the actual value A, proceed as follows:

1. Double click in the device view under the "Parameter setting" module.
2. Navigate in the project view to "Controller > Channel (No.) > Controller difference".
3. Select the corresponding source in the "Manipulated value" area. The following selections are available for selection:

   - Zero
   - Prepared analog input
4. Save the entered parameters with "Project > Save".

#### Realizing three component controller (S7-300, S7-400)

##### Three component controller

For the three component controller structure, the effective actual value is formed by adding the three actual values A, B and C and through adding an adjustable offset. The actual values B and C can be additionally evaluated through factors.

##### Select controller type

Proceed as follows to implement a three-term controller:

1. Double click in the device view under the "Parameter setting" module.
2. Navigate in the project view to "Controller > Channel (No.)".
3. Select the option "Three component controller" in the "Controller type" area.
4. Save the entered parameters with "Project > Save".

##### Prepare setpoint

To set how the setpoint should be prepared, proceed as follows:

1. Double click in the device view under the "Parameter setting" module.
2. Navigate in the project view to "Controller > Channel (No.) > Controller difference".
3. In the "Setpoint source" area, select from which source the setpoint should be referenced.
4. Enter a value in the "safety setpoint" area that should serve as a safety value in the case of a CPU-error and during start-up.
5. Select the behavior of the controller module that should apply as "reaction to CPU error" and for "start-up behavior". The selection consists of:

   - Setpoint = last valid setpoint: upon the occurrence of the event, the setpoint from the last valid setpoint will be assigned. The last valid setpoint remains constant.
   - Setpoint = safety setpoint: upon the occurrence of the event, the setpoint will be assigned the safety setpoint. In this way, a secured emergency mode can be realized.
6. In the "Setpoint ramp" area, set the rise time that should be held with a change of the setpoint in order to reduce an overshooting.
7. In the "setpoint limit" area, set both an high and low limit for the setpoint.
8. Save the entered parameters with "Project > Save".

##### Select signal source for actual value A, B and C

To set the signal sources for the actual values A, B and C, proceed as follows:

1. Double click in the device view under the "Parameter setting" module.
2. Navigate in the project view to "Controller > Channel (No.) > Controller difference".
3. In the "Manipulated value" area, select the respective signal source for each actual value. Here you can select if you want to use no signal ("zero") as a source or if you want to use prepared analog inputs.
4. Save the entered parameters with "Project > Save".

##### Enter factors for actual values B and C and offset

To set the signal sources for the actual values A, B and C, proceed as follows:

1. Double click in the device view under the "Parameter setting" module.
2. Navigate in the project view to "Controller > Channel (No.) > Controller difference".
3. In the "Add" area, set the required weight factor for the actual values B and C.
4. Set the required "offset" that should be added to the actual values.
5. Save the entered parameters with "Project > Save".

4. Save the entered parameters with "Project > Save".

##### Enter multiplier

To set the signal sources for the actual values A and D, proceed as follows:

1. Double click in the device view under the "Parameter setting" module.
2. Navigate in the project view to "Controller > Channel (No.) > Controller difference".
3. Select the desired offset in the "Multiply" area.
4. Save the entered parameters with "Project > Save".

For ratio controllers, the actual value A is used as the controller size, the actual value D as the ratio size. The setpoint input serves as the ratio factor. Through multiplication with the actual value D and addition of an adjustable offset, an effective setpoint is prepared. If the actual value D is turned off, then only the offset will be added to the setpoint.

#### Setting alarm generation (S7-300, S7-400)

##### Procedure

Proceed as follows to set interrupt generation:

1. Double click in the device view under the "Parameter setting" module.
2. Navigate in the project view to "Controller > Channel (No.) > Controller difference".
3. Select the required options in the "alarm generation" area:

   - At "Limit monitor for", select for which signal the alarm should be set.
   - Under "Alarm high limit" you can set from which value the alarm should be activated upon reaching the high limit.
   - Under "Warning high limit" you can set from which value an alarm should be activated upon reaching the high limit.
   - Under "Warning low limit" you can set from which value an alarm should be activated upon reaching the low limit.
   - Under "Alarm low limit" you can set from which value the alarm should be activated upon reaching the low limit.
   - To avoid flickering on the monitoring displays, a hysteresis can be set under "Hysteresis".
4. Save the entered parameters with "Project > Save".

##### Procedure

#### Reverse controller effect (S7-300, S7-400)

##### Principle

Reversing the controller from the assignment "increasing controller difference = increasing manipulated variable" to "increasing controller difference = decreasing manipulated variable" can be done by setting a negative proportion value (GAIN).

##### Procedure

To reverse the controller effect, proceed as follows:

1. Double click in the device view under the "Parameter setting" module.
2. Navigate in the project view to "Controller > Channel (No.) > Controller algorithm".
3. Set a proportion value with the reverse prefix in the "General" area.

   If a positive proportion value is specified and the controller effect should be reversed, set a negative proportional value.
4. Save the entered parameters with "Project > Save".

#### Requirements on process with fuzzy temperature controller  (S7-300, S7-400)

##### Requirement

The process should fulfill the following requirements in order to achieve optimal control using the fuzzy temperature controller:

- At bath heatings the liquid to be heated has to be mixed thoroughly.
- In case of bath-in-bath control systems both liquids have to be mixed thoroughly. At the same time good heat transitions between all the heat-transferring media have to be ensured. In the case of materials with poor heat-transferring properties large transfer surfaces should ensure good heat transportation.
- In case of room temperature control systems thorough mixing (for example with fans) has to be ensured.
- The controlling system gain may not exceed the factor of 3.
- The delay time may not exceed 3% of the recovery time.
- The temperature to be controlled should change by a maximum of 1‰ of the specified maximum temperature at the maximum manipulated value output within the sampling time of the controller.

##### Classification of the Controlled Systems

A controlled system or a process to be controlled is characterized by parameters such as the heat output, the heating mass or the heating capacity of the medium to be heated. With regard to the fuzzy controller a difference is made between "critical" and "non-critical" temperature controlled systems as follows: The control system becomes increasingly critical:

- The greater the heat output,
- the greater the heating capacity of the heating,
- The lower the heating capacity of the medium to be heated,
- the greater the heat transition resistance,
- the smaller the heat transfer surface.

After a manipulated value step change has been applied to the controlled system, it reacts with a step response. The controlled system can also be classified on the basis of this step response: The control system becomes increasingly critical the greater the ratio t<sub>u</sub> / t<sub>g</sub> is and the greater the controlled system gain is. With t<sub>u</sub> / t<sub>g</sub> < 1/10 you have a non-critical control section.

#### Process of fuzzy system identification (S7-300, S7-400)

##### Process of parameter tuning with fuzzy temperature controller

Parameter optimization is based on a self-optimizing fuzzy controller.

The identification of the controlled system begins with a monitoring phase during which no heat output takes place. The duration of the monitoring phase is as follows:

- Monitoring phase continuous controller: Approx. 1 min
- Monitoring phase step controller: Approx. 1 min + 1/2 x actuating time of final control element

This time is used in order to determine temperature trends in the heating medium. Afterwards the maximum heating output of 100% is output. This is visible at the LMN output parameter in the instance DB of the PID_FM instruction.

The range of the first 4% of the temperature increase of the setpoint value range is used for the identification, whereby information about the process response is obtained at 1% and 4% respectively of the temperature increase on the basis of the time that has passed.

The identification is complete when the heating output becomes less than 100%. This behavior can be used, for example, to deactivate the identification via the user program. After the identification phase the controller continues to operate with the determined parameters.

If the setpoint value step change is not adjusted and if the heating output remains permanently at zero, the identification has been terminated unsuccessfully, meaning that the controller cannot control the connected controlled system.

##### Controller behavior with different control sections

Problems do not arise in case of an "uncritical" controlled system, neither during identification nor during controlling.

The identification of a controlled system that is "too critical" is cancelled. Controlling of an identified "critical" controlled system is carried out very "carefully" or slowly.

#### Performing fuzzy system identification (S7-300, S7-400)

##### Requirement

You have selected the "Fuzzy controller" control algorithm during parameter assignment.

You have to ensure as far as possible that the controlled system has settled (no heating-up or cooling-down process) or is changing slowly and monotonously before the identification is carried out. The criterion can be that the temperature change is to approach a straight during a period of one minute. At faster processes this requirement is particularly relevant.

Since the manipulated variable zero is output for approx. one minute by the controller at the beginning of identification, the temperature to be controlled has to lie near the ambient temperature.

##### Starting fuzzy system identification

Follow these steps to tune a fuzzy controller:

1. Select the instance DB of the channel you want to tune.
2. Click the "Start" icon.

   If there is no online connection, this will be established. The current values for the setpoint, process value, and manipulated variable are recorded.
3. Enter a new setpoint (SP_OP) in the "Tuning" group.

   The new setpoint must meet the following conditions:

   - Setpoint step change > 5%
   - New setpoint > process value + setpoint high limit x 0.12
4. Click ![Starting fuzzy system identification](images/23498796811_DV_resource.Stream@PNG-de-DE.png) "Fuzzy process identification" in the "Tuning" group.

##### Result

The new setpoint (SP_OP), SP_OP_ON = TRUE and FUZID_ON = TRUE are sent to the FM. Starting the fuzzy system identification. The IDSTATUS parameter of the CH_DIAG instruction provides information on the identification state. You see in the status display field if a fuzzy system identification is activated or running.

##### Deactivating fuzzy system identification

As long as the bit FUZID_ON = TRUE, the next identification is started whenever the setpoint step change is sufficiently large.

Proceed as follows to turn off the tuning mode:

1. Enter FUZID_ON in a watch table.
2. Set FUZID_ON = FALSE.

**Note**

Deactivation of the tuning mode FUZID_ON = FALSE before identification has been completed does not stop the identification. An identification process that has been started continues to run under all circumstances – with the exception of a negative setpoint value step change.

##### Repeating Fuzzy system identification

A renewed identification is carried out with each additional setpoint step change ≥12% unless the identification has been deactivated again with FUZID_ON = FALSE.

It is also possible to restart the identification by reducing and then increasing the setpoint by 12> %.

##### Canceling fuzzy system identification

Identification can be canceled in the following cases:

- By the controller if a "critical" controlled system is identified. After canceling the controller is in the error state. This state is indicated by the fact that the manipulated variable is reset permanently by the controller. This is also not changed by deactivating the optimization. The error state is deleted by starting a new identification.

  Information about the state of the identification is available through the IDSTATUS parameter of the CH_DIAG FB. More information on this can be found at the end of this section under "Controller State Information".
- By the operator by generating a negative setpoint value step change, whereby the setpoint value must lie below that of Condition 2 (see above "How To Start identification").

Proceed as follows to cancel tuning:

1. Enter a new setpoint that is less than the current process value and meets the same conditions as for the start of fuzzy system identification.
2. Click ![Canceling fuzzy system identification](images/23498796811_DV_resource.Stream@PNG-de-DE.png) "Fuzzy process identification" in the "Tuning" group.

### Measuring the motor transition time (S7-300, S7-400)

#### Introduction

The motor transition time has to be as accurate as possible for good controller results. The data in the actuator documentation contains average values for this type of actuator. The value for the specific actuator used may differ.

You can measure the motor transition time during commissioning if you are using actuators with position feedback or endstop signals. The motor transition time cannot be measured if neither position feedback nor endstop signals are available.

#### Requirement

- The configured motor transition time must be half the size of the real transition time. The actuator is moved for a maximum of twice the transition time in one direction.
- The position feedback must be interconnected for actuators with analog position feedback.
- The endstop signals must be interconnected for actuators with endstop signals.

#### Actuators with analog position feedback

Proceed as follows to measure motor transition time with position feedback:

1. Select the instance DB for which you want to measure the motor transition time.
2. Select the "Use position feedback" check box.
3. Enter the position to which the actuator is to be moved in the "Target position" input field.

   The current position feedback (starting position) will be displayed. Click the ![Actuators with analog position feedback](images/23498796811_DV_resource.Stream@PNG-de-DE.png) "Start transition time measurement" icon.

#### Result

The actuator is moved from the starting position to the target position. Time measurement starts immediately and ends when the actuator reaches the target position. The motor transition time is calculated according to the following equation:

Motor transition time = (output value high limit – output value low limit) × Measuring time / AMOUNT (target position – starting position).

The progress and status of transition time measurement are displayed. The measured transition time is displayed in the field "Measured transition time". Once transition time measurement is complete, the manual mode of the transition signals is switched on (LMNSOPON = TRUE). The actuator will stop being moved.

#### Actuators with endstop signals

Proceed as follows to measure motor transition time with endstop signals:

1. Select the instance DB for which you want to measure the motor transition time.
2. Select the "Use actuator endstop signals" check box.
3. Click the ![Actuators with endstop signals](images/23498796811_DV_resource.Stream@PNG-de-DE.png) "Start transition time measurement" icon.

#### Result

The actuator is moved first to the low endstop, then to the high endstop and then back to the low endstop. Time measurement will start once the actuator has reached the lower endstop and will end when the actuator reaches the lower endstop for the second time. The motor transition time is equal to the time measured divided by two.

The progress and status of transition time measurement are displayed. The measured transition time is displayed in the field "Measured transition time". Once transition time measurement is complete, the manual mode of the transition signals is switched on (LMNSOPON = TRUE). The actuator will stop being moved.

#### Loading measured transition time

1. Click ![Loading measured transition time](images/46478751371_DV_resource.Stream@PNG-de-DE.png) "Transfer online data to an offline project" to save the measured motor transition time locally in the project.

   The measured transition time is saved in the parameter setting of the FM.
2. Click ![Loading measured transition time](images/31289963147_DV_resource.Stream@PNG-de-DE.png) "Download to device" to activate all changes the next time you start the CPU.

### Triggering diagnostic interrupts (S7-300, S7-400)

#### What is a diagnostic interrupt?

If the user program reacts to an internal or external error, you can configure a diagnostic interrupt that interrupts the cyclic program of the CPU and calls the diagnostic interrupt OB, OB 82.

#### Which events can trigger a diagnostic interrupt?

The list shows which events can trigger a diagnostic interrupt:

- The module has not been configured or is incorrectly configured
- Module defective
- Wire break with analog inputs (only 4 to 20 mA)
- High limit violated and low limit violated with analog inputs
- Load break and short circuit with analog outputs

#### Default setting

The diagnostic interrupt is blocked by default.

#### Enabling the diagnostic interrupt

In the "Basic parameters" screen you can block or enable the diagnostic interrupt for the module.

#### Responses to an interrupt-triggering event

The following happens when an event occurs that could trigger a diagnostic interrupt:

- The diagnostic information is stored in the diagnostic records DS0 and DS1 on the module.
- The group error LED lights up.
- The diagnostic interrupt OB (OB 82) is called.
- The diagnostic record DS0 is entered in the start information of the diagnostic interrupt OB.
- If there is no hardware fault, the module continues to control.

If no OB 82 is programmed, the CPU goes to STOP.

#### Diagnostic data record DS0 and DS1

The information as to which event has triggered a diagnostic interrupt is stored in the diagnostic data records DS0 and DS1. The diagnostic data record DS0 contains four bytes, the DS1 16 bytes, whereby the first four bytes are identical to those of the DS0.

### Diagnostic data records DS0 and DS1 (S7-300, S7-400)

#### Read data record from the module

The diagnostic data record DS0 is automatically transferred to the start information when the diagnostic OB is called. There these four bytes are stored in the local data (byte 8 - 11) of the OB 82.

The diagnostic data record DS1 (and hence also the content of the DS0) can be read from the module using the "CH_DIAG" instruction. This only makes sense if an error is reported in a channel in the DS0.

The "CH_DIAG" instruction must be called in the same OB as the FB "PID_FM". This is achieved by the following measure: Set a bit while the OB 82 is being executed. Query this bit in the OB in which the FB "PID_FM" is called and then call the "CH_DIAG" instruction with a set bit.

#### How does the diagnosis text appear in the diagnostics buffer?

If you want to enter the diagnostics message in the diagnostics buffer, call the "WR_REC" instruction "Enter user-specific message in the diagnostics buffer" in the user program. The event number of the respective diagnostics message is specified in the input parameter EVENTN. The interrupt is entered in the diagnostics buffer defined as x = 1 for incoming and x = 0 for outgoing. The diagnostics buffer contains the relevant diagnostics text in the "Meaning" column as well as the time of the entry.

#### Assignments of the Diagnostics Record DS0 in the Start Information

The following table shows the assignments of the diagnostics record DS0 in the start information. All unlisted bits are not significant and are set to zero.

Assignments of diagnostics record DS0

| Byte | Bit | Meaning | Remark | Event No. |

### Measuring transducer fault (S7-300, S7-400)

#### Faults at Measuring Transducers

The following measuring transducer faults can be recognized by the controller module:

- Measuring range violation (underrange)
- Measuring range violation (overrange)
- Wiring breakage (not at all measuring ranges)

If one of these faults occurs, the group error bit "External error" is set in the diagnostics record DS0 and the channel-specific error bits in the diagnostics record DS1 (refer to the tables in the previous section). When these faults disappear, the corresponding bits are reset.
