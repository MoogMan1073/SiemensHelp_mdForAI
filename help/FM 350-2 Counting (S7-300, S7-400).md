## CNT2WRPN (S7-300, S7-400)

### Description

You can use the CNT2WRPN instruction to load the counters and comparators of the counter module using write jobs. To do this, you need to call the CNT2WRPN instruction once for each module.

You should integrate the CNT2WRPN instruction in the user program only if it is necessary to reload counters and comparators in runtime.

### Operating principle

The CNT2WRPN instruction executes the [write command](#write-job-s7-300-s7-400) (JOB_WR) from the [counter DB](#counter-data-block-s7-300-s7-400), transfers the corresponding data from the counter DB and indicates the[status of the write command](#status-of-the-write-job-s7-300-s7-400). The write command is handled internally with the WRREC instruction and the return code is passed to the WRREC instruction.

### Call

The CNT2WRPN instruction can be called in the cycle or optionally in a time-controlled program. The call is not allowed in an event-driven interrupt program.

Before executing write jobs, you must assign the appropriate values in the data area associated with the write command. The last write command must be completed, which means that JOB_WR.NO (data byte DBB0) is deleted from the [counter DB](#counter-data-block-s7-300-s7-400).

### Parameters

| Parameters | Declaration | Data type | Description | The user ... | The instruction ... |
| --- | --- | --- | --- | --- | --- |
| DB_NO | INPUT | WORD | Number of the counter data block | Enters this | Queries this |
| RET_VAL | OUTPUT | INT | Return code of the instruction WRREC | Queries this | Enters this |

## CNT2RDPN (S7-300, S7-400)

### Description

You can use the CNT2RDPN instruction to read the counter values and measured values of the counter module using read jobs. Call the CNT2RDPN instruction cyclically once for each module to do this.

Do not integrate the CNT2RDPN instruction in your user program unless you are not processing read jobs.

### Operating principle

The instruction executes the [read command](#read-job-s7-300-s7-400) (JOB_RD) from the [counter DB](#counter-data-block-s7-300-s7-400), transfers the corresponding data to the counter DB and indicates the [status of the read command](#status-of-the-write-job-s7-300-s7-400-1). The read command is handled internally with the RDREC instruction and the return code is passed to the RDREC instruction.

### Call

The CNT2RDPN instruction can be called in the cycle or optionally in a time-controlled program. The call is not allowed in an event-driven interrupt program.

The last read command must be completed, which means that JOB_RD.NO (data byte DBB2) is deleted from the [counter DB](#counter-data-block-s7-300-s7-400).

## CNT2_CTR (S7-300, S7-400)

### Description

The CNT2_CTR instruction is used to control the digital outputs (enable and disable) and software gates of the counter module. In addition, you receive the feedback signals of the module.

### Operating principle

The CNT2_CTR instruction

- initializes the [counter DB](#counter-data-block-s7-300-s7-400).
- Transfers the counter signals from the counter DB ([CONTROL_SIGNALS](#control_signals-s7-300-s7-400) structure) module.
- Reads the feedback signals The instruction in the counter DB saves the read values to the [CHECKBACK_SIGNALS](#checkback_signals-s7-300-s7-400) structure.

### Call

You can use the CNT2_CTR instruction cyclically (in OB1 or in the cyclic interrupts) for each module. Calling in an interrupt program is not permitted.

Before calling the CNT2_CTR instruction, enter the current control signals in the CONTROL_SIGNALS structure in the [counter DB](#counter-data-block-s7-300-s7-400). When the call of the CNT2_CTR instruction is completed, the feedback signals in the CHECKBACK_SIGNALS structure are updated in the counter DB for further processing.

| --- | --- | --- | --- | --- | --- |
| DB_NO | INPUT | WORD | Number of the counter data block | Enters this | Queries this |

## DIAG_RD (S7-300, S7-400)

### Description

The DIAG_RD instruction can be used to load the data of diagnostic interrupts in the counter DB.

### Operating principle

The DIAG_RD instruction reads 16 bytes of diagnostic data from the counter module and enters these data in the[counter DB](#counter-data-block-s7-300-s7-400) in the data area DIAGNOSTIC_IN_INFO. These data contain the diagnostic status of the entire module (across all counters). Diagnostic data is read internally at the call of the RDSYSST instruction. The return code of this instruction is passed as RET_VAL.

### Call

The DIAG_RD instruction can be called only in the interrupt program.

### Counter data block (S7-300, S7-400)

#### Purpose

The counter DB is the data interface between the user program and FM 350‑2. It contains and applies all data necessary to control and operate the module.

Before the parameter assignment of the module, you must provide the following valid data to the counter DB.

- Module address (address 12.0)
- Channel start address (address 14.0)
- DS-OFFSET (address 18.0), set to 0

### Status of the write job (S7-300, S7-400)

#### Status of a write job

The status of a write job is indicated in the counter DB (DBB1 data byte).

| Bit in JOB_WR  (DBX1.) | Meaning |
| --- | --- |
| .BUSY, 0 | = 1: Write job active. The CNT2WRPN instruction sets this bit as soon as you execute a write job (JOB_WR.NO &gt; 0 and JOB_WR.IMPOSS = 0). The CNT2WRPN instruction deletes this bit as soon as the write job is completed (JOB_WR.NO = 0). |
| .DONE, 1 | = 1: Write job finished. The CNT2WRPN instruction sets this bit as soon as a write job is completed (regardless of errors). The CNT2WRPN instruction deletes the bit when a new write job begins. You can also delete this bit in the user program. |
| .IMPOSS, 2 | = 1: The write job cannot be processed (counter module parameters are not assigned, the call is active or programming device (PG) mode is active). You can leave the write job (JOB_WR) pending or delete it. The CNT2WRPN instruction deletes the bit if the conditions indicated above are fulfilled. |
| .UNKOWN, 3 | = 1: Write job unknown. Your specified write job (JOB_WR) is not in the valid range (see the error message). The CNT2WRPN instruction deletes this bit as soon as a valid number is entered in JOB_WR. The unknown number is retained until this happens. |

Error events are indicated in the binary result bit (BR = 0). Possible errors are:

- Unknown write job (see JOB_WR.UNKOWN).
- Error in data transmission with the WRREC instruction. Error information is available at the RET_VAL output parameter.
- The module checks for data errors in the transmitted data transferred and interprets the data. If a data error occurs, the CHECKBACK_SIGNALS.DATA_ERR bit is set to 1 in the counter DB.
