#### Description

| "OR logic operation" |  |  |
| "EXCLUSIVE OR logic operation" |  |  |
| Effect instruction | "Assignment" | The box is extended with an output.  By extending an instruction box with an output, you can set the signal states of several operands. |
| "S - Set output of the effect to 1" |  |  |
| "R - Set output of the effect to 0" |  |  |

#### &: AND logic operation (S7-1200, S7-1500)

##### Description

You can use the instruction "AND logic operation" to query the signal states of two or more operands and evaluate them according to the AND truth table.

#### X: EXCLUSIVE OR logic operation (S7-1200, S7-1500)

##### Description

You can use the instruction "EXCLUSIVE OR logic operation" to query the signal states of two or more operands and evaluate them according to the EXCLUSIVE OR truth table.

If the signal state of one operand is "1" and the signal state of the other operand is "0", the condition is satisfied. If the signal state of both the operands is either "1" or "0", the condition is not satisfied and the cause is inactive.

When more than two operands are queried, the RLO is "1" if an odd number of the queried operands returns the result "1".

##### Parameters

![Example](images/138329906699_DV_resource.Stream@PNG-de-DE.png)

The cause becomes active when the signal state of one of the two operands is "1". When both operands return the signal state "1" or "0", the cause is inactive.

#### OnDelay: Delay activation (S7-1200, S7-1500)

##### Description

The instruction "OnDelay: Delay activation" delays the activation of a cause by the programmed duration "OnDelay".

The duration "OnDelay" starts when the condition programmed in the cause is met. When the duration has elapsed, the cause becomes active and the signal state at the output of the cause is "1".

| Symbol | Meaning |
| --- | --- |
|  | **Danger** |
| **Danger when reinitializing the actual values**  Reinitializing the actual values of a timer while the time measurement is running disrupts the function of the IEC timer. Changing the actual values can result in inconsistencies between the program and the actual process. This can cause serious damage to property and personal injury.   The following functions can cause the actual values to be reinitialized:  - Loading the block with reinitialization - Loading snapshots as actual values - Controlling or forcing the actual values - The "WRIT_DBL" instruction   Before you execute these functions, take the following precautions:  - Make sure that the plant is in a safe state before you overwrite the actual values. - Make sure that the timer has expired before initializing its actual values. - If you overwrite the actual values with a snapshot, make sure that the snapshot was taken at a time when the system was in a safe state. - Make sure that the program does not read or write the affected data during transmission. |  |

##### Parameters

The following table shows the parameters of the instruction

| Parameter | Declaration | Data type | Memory area | Description |
| --- | --- | --- | --- | --- |
| OnDelay | Input | TIME | I,Q, D, L or constant | Duration by which the activation of the cause is delayed. |

#### Description

You can use the "Assignment" instruction to set a specified operand. If the signal state at the input of the effect returns "1", the specified operand is set to signal state "1". If the signal state at the input of the effect is "0", the specified operand is reset to "0".

### N: Set as long as the cause is active (S7-1200, S7-1500)

#### Description

The action "N: Set as long as the cause is active" sets the signal state at the input of an effect to "1". The signal state remains "1" as long as the assigned cause is active. When the cause becomes inactive, the signal state is set to "0" again.

#### Example

The following example shows how the action works. The example shows three consecutive cycles. The effects were aligned horizontally for simplification.

| --- | --- |

| Cycle 2 | The signal state at the output of the cause changes to "1".   The input of the effect is now also signal "1" and the operand "#myOutput1" is set.    ![Example](images/139982152075_DV_resource.Stream@PNG-de-DE.png) |
| Cycle 3 | The signal state at the output of the cause changes back to "0".   The input of the effect is now also signal "0" again and the operand "#myOutput1" is reset.    ![Example](images/139982148363_DV_resource.Stream@PNG-de-DE.png) |
