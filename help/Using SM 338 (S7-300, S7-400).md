## Terminal assignment of the front connector of SM 338 (S7-300, S7-400)

### Wiring rules

Important rules to observe when wiring the module:

- Ground of the encoder supply is connected to CPU ground potential. Thus, establish a low impedance connection between pin 2 of SM 338 (M) and CPU ground.
- Always use shielded twisted-pair cable to wire encoder signals (pins 3 to 14.) Connect both ends of the shielding.

  You use the shield contact element for the shield contact at the SM 338 (article number: 6ES7390-5AA00-0AA0).
- Connect an external power supply if the maximum output current (900 mA) of the encoder supply is exceeded.

### Operating mode (S7-300, S7-400)

This section contains information on the following topics:

- [Enabling Fast Mode (S7-300, S7-400)](#enabling-fast-mode-s7-300-s7-400)

#### Enabling Fast Mode (S7-300, S7-400)

##### Fast Mode

Fast Mode is enabled in the respective check box.

In Fast Mode, the module operates with faster encoder value detection and compressed feedback interface. The Freeze function is not possible.

#### Isochronous mode (S7-300, S7-400)

##### Hardware requirements

To operate SM 338 in isochronous mode, you need:

- a CPU which supports isochronous mode
- a DP master which supports the constant bus cycle
- Interface module IM 153-x, which supports isochronous mode

##### Properties of the SM 338 in isochronous mode

SM 338 operates in non-isochronous or isochronous mode, depending on system parameters.

In isochronous mode, data are exchanged between the DP master and SM 338 in synchronism to the PROFIBUS DP cycle.

In isochronous mode, all 16 bytes of the feedback interface are consistent.

If synchronism is lost due to disturbance or failure/delay of Global Control (GC), the SM 338 resumes isochronous mode at the next cycle without error reaction.

If synchronism is lost, the feedback interface is not updated.

> **Note**
>
> For basic information on isochronous mode, refer to the [Isochronous mode function manual](http://support.automation.siemens.com/WW/view/en/15218045).

### Enable (S7-300, S7-400)

#### Enabling diagnostic interrupt (S7-300, S7-400)

##### Diagnostic interrupt

The diagnostic interrupts are not preset - in other words, they are inhibited without appropriate enabling.

Diagnostic interrupts are enabled in the respective check box.

If you have enabled diagnostic interrupts, the incoming error events (initial occurrence of the error) and outgoing error events (message after troubleshooting) are reported by means of [interrupts](#diagnostics-of-the-sm-338-s7-300-s7-400).

The CPU interrupts user program execution, and executes diagnostic interrupt OB.

The diagnostic information is consistent until such time as diagnostic interrupt OB is exited. When diagnostic interrupt OB is exited, the diagnostic interrupt is acknowledged on the module.

#### Selecting the encoder type (S7-300, S7-400)

##### Encoder type

The SM 338 supports the following encoder types:

- Absolute value encoder (SSI), frame length 13 bit
- Absolute value encoder (SSI), frame length 21 bit
- Absolute value encoder (SSI), frame length 25 bit

The encoder type can be found in the technical data of the absolute value encoder used.

The selection "No encoder" switches off the encoder input.

#### Selecting the code type (S7-300, S7-400)

##### Code type

The parameter "[Code type](Motion%20Control%20%28S7-300%2C%20S7-400%29%20%28TFMCMain3_400enUS%29.md#graydual-converter-s7-300-s7-400)" is used to determine the handling of the code supplied by the encoder:

- When "Gray" is set, the encoder value supplied by the absolute encoder in gray code is converted to dual code.
- When "Binary" is set, the supplied encoder value is not converted.

#### Selecting the transmission rate (S7-300, S7-400)

##### Transmission rate

With the "Transmission rate", you can determine the data transmission rate from the SSI encoder to the SM 338.

Select the desired transmission rate in the drop-down list.

Observe the relationship between line length and transmission rate:

- up to 320 m → 125 kHz
- up to 160 m → 250 kHz
- up to 60 m → 500 kHz
- up to 20 m → 1 MHz

#### Selecting monoflop time (S7-300, S7-400)

##### Monoflop time

The monoflop time is the break interval between 2 SSI message frames.

The configured monoflop time must be greater than the monoflop time of the absolute value encoder used (see manufacturer's technical data).

The time 2 × (1 / transmission rate) is added to the configured value. With a transmission rate of 125 kHz and a configured monoflop time of 16 μs, the actual effective monoflop time is 32 μs.

The following limitation applies to the monoflop time of the absolute value encoder:

(1 / transmission rate) < monoflop time of absolute value encoder < 64 μs + 2 × (1 / transmission rate)

Select the corresponding monoflop time in the drop-down list.

The specification of the monoflop time is relevant to [freewheeling encoder value acquistion.](Motion%20Control%20%28S7-300%2C%20S7-400%29%20%28TFMCMain3_400enUS%29.md#encoder-value-acquisition-s7-300-s7-400)

#### Entering number of adjusted bits (S7-300, S7-400)

##### Number of adjusted bits [0..12]

SM 338 requires the number of adjusted bits to determine the encoder position and to correctly detect the direction of movement of the encoder.

This value can be found in the technical data of the absolute value encoder used.

#### Selecting the digital input for the FREEZE function (S7-300, S7-400)

##### FREEZE function

The FREEZE function "freezes" the actual encoder values of SM 338. The FREEZE function is coupled to the digital inputs DI 0 and DI 1 of SM 338.

The freeze function is triggered by a signal transition (positive edge) at digital input DI 0 or DI 1. Bit 31 = 1 (output address) identifies a frozen encoder value. You can freeze one, two or three encoder values using one digital input.

To allow their event-triggered evaluation, the encoder values are retained until the FREEZE function is terminated.

##### Enabling the FREEZE function

The FREEZE function must be enabled during parameter assignment.

To do so, select the digital input at which a rising edge freezes the encoder value:

- off (encoder value is not frozen)
- DI 0
- DI 1

##### Canceling the FREEZE function

The FREEZE function must be terminated separately at each encoder input. You acknowledge the function in the user program by using the operation T POB "xyz" to set bit 0, 1 or 2 according to the encoder input (see the [program example](#addressing-sm-338-s7-300-s7-400)).

The acknowledgement resets bit 31 of the corresponding encoder value, and initiates a refresh of the encoder values. You can freeze the encoder values again after you cleared the ACK bit at the output address of the module.

In isochronous mode, the acknowledgement is processed at the time T<sub>o</sub>. From this point on, you can freeze the encoder values again by setting the digital inputs.

> **Note**
>
> The FREEZE function is automatically acknowledged if you reconfigure the corresponding encoder input with different parameters.  
> I The freeze function will be unaffected if the parameters are identical.

### Diagnostics of the SM 338 (S7-300, S7-400)

The SM 338 provides diagnostics messages, i.e. it always provides all diagnostics messages without user intervention.

#### Measures after a diagnostic message

Actions initiated by diagnostic messages:

- The diagnostic message is entered in the diagnosis of the module and forwarded to the CPU.
- The SF LED on the module is lit.
- If you configured "[Enable Diagnostic Interrupt](#enabling-diagnostic-interrupt-s7-300-s7-400)", the system triggers a diagnostic interrupt and calls the diagnostic interrupt OB.

#### Reading diagnostic messages

You can read detailed diagnostic messages using instructions in the user program.

The error cause can be displayed in the module diagnostics.

#### Diagnostic message using the SF LED

The SM 338 indicates errors at its SF LED (group error LED.) The SF LED lights up when the SM 338 generates a diagnostic message. It goes dark after all error states are cleared.

The SF LED also lights up to indicate external errors (short circuit at the encoder supply), regardless of the CPU operating state (at POWER ON.)

The SF LED lights up temporarily at startup during the self-test of SM 338.
