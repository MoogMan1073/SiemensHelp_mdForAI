> **Note**
>
> The circuits for the counter inputs (encoder supply, encoder signals) are non-isolated from the ground of the CPU. You should therefore connect terminal 4 (1M) to the CPU ground using a low-resistance connection.
>
> If you supply the encoders with external voltage, you must also connect the mass of this external voltage supply to the ground of the CPU.

You will find more information on construction and wiring in the [manual](http://support.automation.siemens.com/WW/view/en/1118412) for the FM 450-1 counter module.

### Auxiliary voltage 1L+, 1M

Connect a direct voltage of 24 V to the 1L+ and 1M terminals for the 5 V and 24 V encoder voltage supply.

An integrated diode protects the module from a reversal in the polarity of the auxiliary voltage.

The module monitors whether the auxiliary voltage is connected.

### 5.2 V DC encoder power supply

The module generates a voltage of 5.2 V from the auxiliary voltage 1L+/1M at a maximum current of 300 mA per count channel; this is provided at the corresponding "DC5.2V" connection for the short-circuit proof supply of a 5 V encoder.

### 24 V DC encoder power supply

1L+/1M is provided at output "DC24V" for 24V short-circuit proof encoder voltage supply. The encoder power supply is short-circuit checked.

### 5 V encoder signals A and /A, B and /B, N and /N

You can connect incremental encoders with 5 V differential signal to each counter in accordance with RS 422, i.e. incremental encoders with the differential signal A and /A, B and /B, N and /N.

The A and /A, B and /B, N and /N signals are connected via the correspondingly labeled terminals

The signals N and /M are only connected if you wish to set the counter to the zero mark of the encoder.

The inputs are not isolated from the S7-400 bus.

### 24 V encoder signals A*, B* and N*

24 V signals are identified with the letters A*, B* and N*.

You can connect three different types of encoders to each counter:

- Incremental encoder with 24 V signals:

  The signals A*, B* and N* are connected via the correspondingly labeled pins.
- Pulse encoders without directional level:

  The signal is connected to terminal A*
- Pulse encoders with directional level:

  The count signal is connected to the terminal A*. The directional level is connected to terminal B*.

The inputs are not isolated from the S7-400 bus.

### Input filter for 24 V encoder signals

To suppress faults, you can configure input filters (RC elements) with a uniform filtering time for the 24 V inputs A*, B* and N*. The following two input filters are available for each counter:

| Characteristics | Input filter 1  (default setting) | Input filter 2 |
| --- | --- | --- |
| Typical input delay | 1 μs | 15 μs |
| Maximum count frequency | 200 kHz | 20 kHz |
| Minimum pulse width of the count signals | 2.5 μs | 25 μs |

### Digital inputs

You can use digital inputs 1I0 and 1I1 to control the gates of counter 1.  
Digital inputs 2I0 and 2I1 are used to control the gates of counter 2.  
The gates can be both level-controlled and edge-controlled.

The 1I2 digital input is for setting counter 1 to the load value.  
Digital input 2I2 is for setting counter 2 to the load value.

The digital inputs are operated with a 24 V nominal voltage.

The digital inputs are isolated from the bus of the S7-400 and the counter inputs.

### Input filter for digital inputs

To suppress faults you can assign parameters to the input filters (RC elements) with a uniform filtering time for the digital inputs 1I0, 1I1 and 1I2 or 2I0, 2I1 and 2I2. The following two input filters are available:

| Characteristics | Input filter 1  (default setting) | Input filter 2 |
| --- | --- | --- |
| Typical input delay | 1 μs | 15 μs |
| Minimum pulse width of input signals | 2.5 μs | 25 μs |

### Digital outputs

FM 450-1 has the digital outputs 1Q0 and 1Q1 (for counter 1) and 2Q0 and 2Q1 (for counter 2) for directly triggering control processes.

The digital outputs are supplied by load voltage 2L+.

The digital outputs are isolated from the bus of the S7-400 and the counter inputs.

The digital outputs are Sourcing outputs and can carry load current of 0.5 A. They are protected against overload and short circuit.

> **Note**
>
> The direct connection of relays and cutouts can be executed without external wiring

The time response of the digital outputs depends on parameter assignment.

### 2L+/2M load voltage

To ensure the power supply for digital outputs 1Q0 and 1Q1 or 2Q0 and 2Q1, you must connect a load voltage of 24 V to the module terminals 2L+ and 2M.

An integrated diode protects the module from reversing the polarity of the load voltage.

The load voltage 2L+/2M is not monitored by the FM 450-1.

#### Selecting an operating mode (S7-300, S7-400)

##### Description

You define counter module functionality by specifying an operating mode.

| Operating mode | Description |
| --- | --- |
| [Count continuously](Counting%20and%20measuring%20%28S7-300%2C%20S7-400%29.md#count-continuously-s7-300-s7-400)  (with or without gate) | The FM 450‑1 counts continuously from the current counter value. |
| [Count once](Counting%20and%20measuring%20%28S7-300%2C%20S7-400%29.md#count-once-s7-300-s7-400)  (with hardware or software gate) | When the gate opens, the FM 450‑1 counts from the load value to the count limit. |
| [Count periodically](Counting%20and%20measuring%20%28S7-300%2C%20S7-400%29.md#count-periodically-s7-300-s7-400)  (with hardware or software gate) | When the gate opens, the FM 450‑1 counts from the load value between the count limits. |

The default setting is the "Count continuously" operating mode.

#### Selecting the Gate Function (S7-300, S7-400)

##### Description

You define the count process in the selected operating mode by specifying a gate control.

- without gate

  All gates are open.
- [Software gate](Counting%20and%20measuring%20%28S7-300%2C%20S7-400%29.md#software-gate-s7-300-s7-400)

  The software gate is opened and closed by the input parameter / control bit SW_GATE.
- [Hardware gate](Counting%20and%20measuring%20%28S7-300%2C%20S7-400%29.md#hardware-gate-s7-300-s7-400), level-controlled or edge-controlled

  The hardware gate is opened and closed by the corresponding signals at the digital inputs.

The default setting is "Without gate".

#### Selecting the counting range (S7-300, S7-400)

##### Description

There is a 32-bit wide count register on the module. With the Count range, you select whether the module is to count only in the positive range or whether the 32nd bit is interpreted as a sign bit thus allowing negative numbers to be represented.

Select the desired counting range.

For more information, please see "[Counting ranges and count limits](Counting%20and%20measuring%20%28S7-300%2C%20S7-400%29.md#counting-basics-s7-300-s7-400)".

##### Violation of the high limit, low limit and zero pass

A bit is set in the DB of the CNT_CTRL instruction for both count limits when the high or low limit is violated.

A bit is also set in the DB in the "-31 to +31-bits" counting range in the event of a zero pass.

Depending on the count direction, violation of the high or low limit is also indicated in the "0 to +32 bits" counting range upon zero pass.

| Event | Status bit in DB |
| --- | --- |
| Violation of the high limit | STS_OFLW is set. |
| Violation of the low limit | STS_UFLW is set. |
| Zero pass | STS_ZERO is set. |

##### Initiating Hardware Interrupts

You can also signal the violation of the high limit and low limit and the zero pass events using hardware interrupts.

##### Setting the counter (S7-300, S7-400)

###### Overview

If you want to start the count from a specific value (the load value), you must parameterize the signal that is to be used to set this counter to the load value. You can set the counter as follows:

- With the L_DIRECT input parameter of the CNT_CTRL instruction
- With an external signal, using either the 1I2 (2I2) input or the input in combination with the encoder zero mark

###### Load value

Any number within the limits of the count range can be set for the count range.

The load value is interpreted according to the selected counting range. If, for example, you specify FFFF FFFF H as the load value, this will be interpreted as 4 294 967 295 in the "0 to +32 bits" counting range and as -1 in the counting range "-31 to +31 bits".

Enter the load value in the DB of instruction CNT_CTRL.

The following ranges of values are permitted for the load value:

| Range of values for load values |  |
| --- | --- |
| Low limit | Maximum low count limit |
| High limit | Maximum high count limit |

###### Setting the Counter with an External Signal

You can select two different external signals with which you can set a counter to the load value:

- [Input 1I2 (2I2)](#setting-the-counter-with-input-1i2-2i2-s7-300-s7-400) only
- [Input 1I2 (2I2) and the zero mark](#setting-the-counter-with-input-1i2-2i2-and-the-zero-mark-s7-300-s7-400) of the encoder

You use the zero mark of the encoder if you want to synchronize the counter to a specific counter status at a specific point in your process. This improves the precision of the counting process.

The counter is set independently of the Count mode.

When the counter has been set with an external signal, bit STS_SYNC will be set in the DB.

> **Note**
>
> Synchronization of a counter with zero mark only makes sense if the gate is open.
>
> If you have only enabled one count direction when setting a counter with an external signal, you must note that when the gate is closed the current count direction is saved (frozen). This enables synchronization of the counter against the enabled count

##### Setting the counter with input 1I2 (2I2) (S7-300, S7-400)

###### Setting the counter with input 1I2 (2I2)

The counter can be loaded with the load value by means of a rising edge at input 1I2 (2I2) (DI set).

You can define the behavior of the FM 450‑1 with a rising edge at input 1I2 (2I2) with the variables ENSET_UP and ENSET_DN in the DB of the CNT_CTRL instruction and by assigning parameters.

| Parameter | Behavior of the FM 450‑1 |
| --- | --- |
| ENSET_UP set | The counter is only set if counting up. |
| ENSET_DN set | The counter is only set if counting down. |
| ENSET_UP and ENSET_DN set | The counter is set when counting up and counting down. |
| Parameterization "Single setting of counter" | The counter is set only on the first rising edge at input 1I2 (2I2).  If the counter is to be set again, you must first set ENSET_UP or ENSET_DN again. The counter is then set again on the next rising edge at input 1I2 (2I2). |
| Parameterization "Multiple setting of counter" | The counter is set at every rising edge at input 1I2 (2I2) as long as ENSET_UP and/or ENSET_DN is set. |

> **Note**
>

#### Specifying pulse duration (S7-300, S7-400)

##### Pulse duration

You can set a pulse duration in order to adapt to the actuators (contactors, actuators, etc) used in your process. The pulse duration indicates how long outputs Q0 and Q1 are active when a comparison value is reached.

The pulse duration specifies the minimum period for which the output is to be set. This setting is only effective if you preselect the output behavior accordingly. The pulse duration setting is not effective if the output is to be activated at a value between the comparison value and high limit violated or low limit violated.

The pulse duration starts when the output is set. The pulse duration inaccuracy is less than 1 ms.

A value between 0 and 500 ms may be set for the pulse duration. This value applies to both outputs together.

If the pulse duration = 0 ms, the output is set when the comparison value is reached and is reset at the next count pulse.

The value 0 is set by default for the pulse duration.

> **Note**
>
> If you specify a pulse duration of 0 ms, the output will be activated for as long as the counter value is equal to the comparison value (see top of figure below).
>
> Control pulses may be lost at the outputs if the time interval between the count pulses is less than the digital output switching times (up to 300 µs) (see bottom half of figure below).
>
> You should therefore make sure that the interval between the count pulses is greater than the switching times of the digital outputs.

##### Comparison values and outputs (S7-300, S7-400)

###### Description

You can store two comparison values (comparison value 1 and 2) for the counters in the FM 450‑1; these are assigned to the two relevant digital outputs (comparison value°1: 1Q0 or 2Q0, comparison value 2: 1Q1 or 2Q1). The comparison values are compared with the current counter value. When the counter value reaches a comparison value, the relevant output can be set.

###### Comparison Values 1 and 2

You enter the two comparison values in the DB of the CNT_CTRL instruction (CMP_V1, CMP_V2) and transfer them to the FM 450‑1 by setting the bits T_CMP_V1 or T_CMP_V2. This does not affect the count.

The comparison values must be within the limits of the selected count range. The comparison value is interpreted according to the selected count range. If you specify FFFF FFFF H as the comparison value, for example, this value will be interpreted as 4,294,967,295 in the "0 to +32 bits" counting range and as -1 in the"-31 to +31 bits" counting range.

The valid value ranges for the comparison values are as follows:

| Range of values for comparison values |  |
| --- | --- |
| Low limit | Maximum low count limit |
| High limit | Maximum high count limit |

###### Enabling the Outputs

Before the outputs can be controlled, you must first enable them by setting the relevant bits in the DB. If you reset one of these bits the associated output is switched off immediately even if you have parameterized a pulse duration for them.

The following bits must be set before you can enable the outputs:

| Output | ... Is enabled by enable bit |
| --- | --- |
| Q0 | CTRL_DQ0 |
| Q1 | CTRL_DQ1 |

#### Enabling hardware interrupt trigger (S7-300, S7-400)

##### Introduction

You can set which events are to trigger a hardware interrupt for the FM 450‑1 counters.

##### What is a Hardware Interrupt?

If you want there to be a response to a specific event irrespective of the CPU cycle, each FM 450‑1 counter can trigger a hardware interrupt. The CPU interrupts the cyclic program on receiving the interrupts and executes the hardware interrupt OB.

##### Enabling the Hardware Interrupt

Hardware interrupts are enabled in two steps:

1. When configuring the hardware, you enable the interrupts for the module in the basic parameters and choose whether the module is to trigger diagnostic and/or hardware interrupts.
2. You then select the relevant [events](#events-which-can-trigger-a-hardware-interrupt-s7-300-s7-400) under "Enable hardware interrupts".

##### Process interrupt OB, OB 4x

If a hardware interrupt occurs, the user program is interrupted, the data is transferred from the module to the [start information of OB 4x](#ob-4x-start-information-s7-300-s7-400) and OB 4x is called. The process interrupt is acknowledged by exiting the OB 4x.

If no OB 4x is programmed, the CPU goes into STOP mode. If you then switch back to RUN, the hardware interrupt requirements are deleted.

##### Lost hardware interrupt

If an event occurs that is to initiate a hardware interrupt and the same previous event has not yet been acknowledged, no further hardware interrupt is initiated; the hardware interrupt is lost. This may lead to the "Hardware interrupt lost" diagnostic interrupt, depending on the parameterization.

##### Default setting

No hardware interrupts are enabled in the default setting.

#### Events which can trigger a hardware interrupt (S7-300, S7-400)

##### Which events can trigger a hardware interrupt?

The following events can trigger a hardware interrupt (independently of each other for each of the two counters) during operation of the FM 450‑1:

- Opening of the gate (in the operating modes with hardware or software gate)
- Closing of the gate (in the operating modes with hardware or software gate)
- Violation of the high limit
- Violation of the low limit
- Zero pass
- Reaching comparison value 1 or 2 in the up direction
- Reaching comparison value 1 or 2 in the down direction
- Setting the counter (DI set)

Select the check box for the relevant events.

You can select any number of events for triggering a hardware interrupt. Observe the [basic conditions for output behavior](#basic-conditions-for-the-behavior-of-the-digital-outputs-s7-300-s7-400) applicable to hardware interrupts on reaching a comparison value.

#### Triggering diagnostics interrupts (S7-300, S7-400)

##### Definition of a diagnostics interrupt

You can determine reactions to internal or external errors in the user program, by programming a diagnostics interrupt which interrupts cyclic program execution on the CPU, and triggers a call of diagnostics interrupt OB 82.

##### Events which can trigger a diagnostics interrupt

The following events can trigger a diagnostic interrupt:

- Faults in the external auxiliary 1L+/1M voltage.
- Short-circuit or overload at the 5.2 V DC encoder supply
- No module configuration
- Faulty module parameters
- Time monitoring (watchdog) triggered
- RAM defective
- Hardware interrupt lost
- Signal A error (wire break, short-circuit, cable missing)
- Signal B error (wire break, short-circuit, cable missing)
- Signal N error (wire break, short-circuit, cable missing)

##### Enabling diagnostics interrupts

When configuring the hardware, you enable the interrupts for the module in the basic parameters and choose whether the module is to trigger diagnostic and/or hardware interrupts.

##### Reactions to a diagnostics interrupt

When an event occurs that could trigger a diagnostic interrupt:

- The diagnostics information is stored in the [diagnostic data records DS0 and DS1](#diagnostics-data-record-ds0-and-ds1-s7-300-s7-400) in the module.
- The group error LED is lit.

  The group error LED does dark when the error is eliminated.
- Call of the diagnostics interrupt OB 82.
- The diagnostic data record DS0 is entered in the start information of the diagnostic interrupt OB.
- The count continues unchanged if an OB 82 has been programmed. If no OB 82 has been programmed, the CPU will go into STOP mode.

##### Default setting

No diagnostic interrupts are enabled in the default setting.

#### Diagnostics data record DS0 and DS1 (S7-300, S7-400)

##### Diagnostics data record DS0 and DS1

#### Data error (S7-300, S7-400)

##### Data error events

FM 450‑1 checks all new parameters transferred to the module. The module reports any errors returned in this check.

##### Where are data errors displayed?

The [CNT_CTRL](FM%20x50-1%20Counting%20%28S7-300%2C%20S7-400%29.md#cnt_ctrl-s7-300-s7-400) instruction enters the data errors with the error numbers in the CNT_CTRL DB. You can access this data word in the user program using the DA_ERR_W variable.

##### Possible data errors

The table below lists the data errors and their error numbers and meaning.

| No. | Meaning |
| --- | --- |
| 0 | No error |
| 201 | Incorrect receiver selection value |
| 202 | Invalid diagnostics value of the signal pair |
| 203 | Incorrect value for signal evaluation |
| 204 | Incorrect 24 V count signal input filter value |
| 205 | Invalid value at digital input filters |
| 206 | Reversal of direction not allowed |
| 207 | Incorrect 1Q0 (2Q0) behavior configuration |
| 208 | Incorrect 1Q1 (2Q1) behavior configuration |
| 209 | Pulse duration out of limits |
| 211 | Wrong operating mode selected |
| 212 | No gate or both gates defined |
| 215 | A different count direction was selected for configuring output behavior "Active on reaching the comparison value for pulse duration up or down" in parameter assignment for the hardware interrupt "Reaching the comparison value up or down". The directions set must match. |
| 216 | Gate control interrupts are only possible in modes with gate control. |
| 217 | Interrupt on "Reaching comparison values" is not valid for output behavior "On when counter value &gt;= comparison value" or "On when counter value &lt;= comparison value". |

##### How are data errors acknowledged?

Correct the parameter values according to specifications. Transfer the corrected parameter set to the FM 450‑1 again. It will recheck the parameters and delete the data error in the DB.

#### Operator error (S7-300, S7-400)

##### Operator errors events

Operator errors occur as a result of improper operation of the module caused by the incorrect input of control signals.

##### Objects which indicate operator errors

The [CNT_CTRL](FM%20x50-1%20Counting%20%28S7-300%2C%20S7-400%29.md#cnt_ctrl-s7-300-s7-400) instruction enters the operator error numbers in the DB. Instruction CNT_CTRL indicates an operator error has occurred when output parameter OT_ERR is set. You can access this data word in your program using the OT_ERR_B variable.

##### Possible operator errors

The table below lists the operator errors and their error numbers and meaning.

| No. | Meaning |
| --- | --- |
| 0 | No error |
| 1 | Operating mode can not be started using the SW gate |
| 2 | Operating mode can not be canceled |
| 4 | Only allowed when the CPU is in STOP |

##### How to acknowledge operator errors

You acknowledge the error with the OT_ERR_A input parameter of the [CNT_CTRL](FM%20x50-1%20Counting%20%28S7-300%2C%20S7-400%29.md#cnt_ctrl-s7-300-s7-400) instruction.
